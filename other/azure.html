Service Principal

  • Service Principal is a reference to an application.
  • It controls what the app can do or who can access the app.
  • This is also same as the Application ID.
# Service principal expiry.
az ad sp credential list --id <app_id> --query "[].endDate" -o tsv

# Renew service principal.
az ad sp credential reset --name <display_name or object_id> --years 2

Account and subscription

# Show my current account information.
az account show

# Set your subscription.
az account set -s <subscription>

AKS (Azure Kubernetes Service)

# Add a cluster information to kube config. "--admin" flag may or may not work depending on your permission in Azure.
az aks get-credentials -n <cluster-name> -g <resource-group> --admin

# Show details of AKS cluster.
az aks list --resource-group <resource-group>
az aks list --resource-group <resource-group> --query id

# Pod identity.
kubectl get azureidentity -n <namespace>
kubectl get azureidentitybinding -n <namespace>

# Check nodepool.
az aks nodepool list --cluster-name <cluster_name> --resource-group <resource_group>
# Azure Container Registry
az acr list

# Inspect ACR and AKS integration
az role assignment list --assignee <app_id> --scope <acr_id>

Pod identity

  • Pods to be authenticated by AAD.
  • Operator creates azureIdentity (with MSI) and azureIdentityBinding. (with Pod label)

Managed Identity Controller (MIC)

  • Deployed as Deployment.
  • Watches azureIdentityBinding objects and creates azureAssignedIdentity.
  • Watches what Azure Identity is bound to Pod and assign corresponding MSI to Node where that Pod is running.

Node Managed Identity (NMI)

  • Deployed as DeamonSet.
  • Requests ADD token on behalf of Pod.

ADLS (Azure Data Lake Storage)

Azure data lake storage is PaaS offering build on Azure BLOB storage.

Azure BLOB storage

  • MS storage for Binary Large Objects (BLOB) that stores Massive unstructured data.
  • Storage account includes containers, which in turn includes BLOBs.
  • Containers must be in lowercase. Blobs can be Block (text or binary) Append (log files) Page (frequent read/write)

Azure Queue storage

  • Similar to Azure Queue storage that stores large number of messages.
  • Storage account includes queues, which in turn includes messages.
  • Queues must be in lowercase. Messages can be any format up to 64KB.

Authentication/Authorization

  • Azure AD group.
  • Service Principal.
  • Managed Identity.

Mount container to virtual machine (NFS v3)

  • Storage Account must be created with NFS v3 enabled.
  • Storage Account -> Networking must have the VNET and subnet of virtual machine(s).
mkdir -p <directory_to_create_mount>
yum install cifs-utils -y
yum install nfs-utils -y
echo "<storage_account_name>.blob.core.windows.net:<storage_account_name>/<container_name> <directory_to_create_mount>    nfs defaults,sec=sys,vers=3,nolock,proto=tcp,nofail    0 0" >> /etc/fstab
mount <directory_to_create_mount>

Mount container to AKS (Blobfuse)

  • Storage Account -> Container must allow read & write.
  • PV and PVC must be created before the pod.

PV

apiVersion: v1
kind: PersistentVolume
metadata:
  name: (volume_name)
spec:
  capacity:
    storage: (volume_size)  # Ex. 30Gi
  accessModes:
    - ReadWriteMany
  persistentVolumeReclaimPolicy: Retain  # Can also set to "Delete"
  storageClassName: (storage_class_name)
  mountOptions:
    - -o allow_other
    - --file-cache-timeout-in-seconds=120
  csi:
    driver: blob.csi.azure.com
    readOnly: false
    volumeHandle: must_be_a_unique_name
    volumeAttributes:
      resourceGroup: (resource_group)
      storageAccount: (storage_account)
      containerName: (container_name)
      AzureStorageAuthType: msi
      AzureStorageIdentityResourceID: (id)  # Should look something like "/subscriptions/../resourceGroups/../providers/Microsoft.ManagedIdentity/userAssignedIdentities/.."

PVC

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: (volume_claim_name)
spec:
  accessModes:
    - ReadWriteMany
  resources:
    requests:
      storage: (volume_size)  # Ex. 30Gi
  volumeName: (volume_name)
  storageClassName: (storage_class_name)

Pod

volumes:
  - name: (any_volume_name)
    persistentVolumeClaim:
      claimName: (volume_name)

volumeMounts:
  - name: (any_volume_name)
    mount_path: (mount_path)
