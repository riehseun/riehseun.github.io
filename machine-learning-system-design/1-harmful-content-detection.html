Harmful Content Detection


# Requirement

Functional
  Where does potentially harmful content exist? Instagram feed
  What is considered as harmful post? Posts with violent or hate text, nude images and videos
  Should the system tell author why content was removed? Yes
  Can users report post as harmful? Yes
  Should the system support multiple languages? Yes

Non-functioanl
  How many posts does each user create? 1 per day 
  How many users are there in platform? 100M daily active users (100M post created per day)
  Is there training data available? Yes, 100M pairs of posts and 1/0 indicating violence/hate/nude or not
  How fast should harmful contents be removed? As soon as possible
  Should model be deployed to user device? No
  Should model be continously trained? Yes

# Problem

ML
  Input - post
  Output - whether post is harmful or not
  Binary classification
  
Business
  Minimize total number of harmful impressions, subject to minimum precision

# Design

Post Upload -> Filter -> Feature Extraction -> Inference -> Remove or Human Review or Allow

# Data

Human labelling to detect violence, hate, nudity
User reports
Results from human review

# Features

Post
  Text - encoder model like DistlimBERT for multi-language
  Text embedded within image - model to extract texts with images, then DistlimBERT
  Image, video, audio - image/video/audio encoder
Author
 Violation history - standardize (Mean 0, Var 1)
 Date of account creation

# Model

Baseline
  Logistic regression - binary classification to determine whether a post in harmful or not

Architecture
  Separate model for violence/hate/nude - lots of training data needed, training cost
  Create one model but with different classification heads for violence/hate/nude

Training
  Binary classification  - use cross-entropy (log-loss)
  Loss - compute individual loss for violence/hate/nude, then create weighted sum of those losses
  
# Evaluation

Offline
  Recall - percentage of actual harmful content that the model was able to retrieve
  Precision - whether predicted harmful contents are correctly classified
  PR-AUC

Online
  Total number of harmful impressions

# Discussion

Filter
  Blacklisted keyword
  Known harmful images

Inference
  Use fast and light model first
  Use slow and heavy model after the post passes the review for double checking

Feedback loop
  Use human review for further training dataset
