Model Serving


# Requirement and Problem

How many models are there? Over 100
What kind of models? Feed recommendation, harmful content detection, product search, fraud detection

Are inferences batch and real-time? Both
What should be latency for real-time inference? Under 200ms
Are models continuously trained? Yes
How many daily active users are there? 1B
How many inference requests does each user make? depends on use case
Should the system monitor model metrics? Yes
Should the system perform model deployment like A/B testing? Yes

ML problem
  Input - requests with static and dynamic features
  Output - prediction
  Platform that serves huge number of API requests, compute/generate huge size of batch predictions

# Detailed Design

Deployment
  Deploy models into different regions such as US East, US West, etc
  This avoids cross-regional latency and single point of failure

Inference
  Cache - serve frequent prediction (For example, use Redis)
  Feature store - stores batch features and its embeddings
  Candidate generation - leverage embeddings from features store, use ANN to generate candidates offline
  Model - use dynamic batching (For example, micro-batching per 5-10ms) to increase throughput

Performance
  Quantization
  Knowledge distillation (only if teacher model is available)

Scalability
  Scale based on usage (For example, kubernetes)
  Scale based on traffic forecast
