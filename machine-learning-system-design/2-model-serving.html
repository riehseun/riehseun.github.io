# Requirement and Problem

How many models are there? Over 100
What kind of models? Feed recommendation, harmful content detection, product search, fraud detection

Are inferences batch and real-time? Both
What should be latency for real-time inference? Under 200ms
Are models continuously trained? Yes
How many daily active users are there? 1B
How many inference requests does each user make? depends on use case

Problem - platform that serves huge number of API requests, compute/generate huge size of batch predictions

# Detailed Design
Horizontally scale API servers
Pre-pull latest models from model registry rather than pulling it during runtime
Before using the model to compute inference, check if user-inference data exists in pre-compute inference cache
When user features are not available at all (users not signed in, anonymous users, users not sharing geo-location) recommend popular ADs
