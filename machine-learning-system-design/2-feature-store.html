Feature Store


# Requirement

Should feature store consist on online and offline store? Yes
Should feature store be used during model training and inference? Yes

How many models should the system support? 100+
How fast should inference be? Under 1s
Should models be continuously trained? Yes
What is size of incoming data per day? 1TB per day
What is rate of growth of data 10% per year

# Design

Data ingestion -> Data processing -> Online and offline feature store -> Feature registry -> Model serving or training

Data ingestion
  Kafka brokers can autoscale based on ingestion load

Data procesing
  Spark structured streaming can be used to tranform features

Online store
  Use Redis for fast read during inference
  Store latest value of features with TTL
  Key should be entity id and value should be feature vector

Offline store
  Use S3 object storage for persistant storage
  Store entire history of features
  
Feature registry
  Use Feast to store metadata such as feature definition, ownership, version

Request -> Online feature store (1-10ms) -> Inference (~500ms) -> Output

# Discussion

Availability
  For online store, create multiple replicas of cache servers in different geo regions
Reliability
  For offline store, partition data into shards, and create mutiple replicas of shards, and spread replicas across different geo region
  If using S3, these will be taken care of
Scalability
  Horizontally scale cache servers (online) and data nodes (offline) based on traffic forecast
Performance
  Keep size of each replicas small and perform parallel read or write
Monitoring
  System metric (QPS, latency, throughput)
  Model metric (feature drift)

Point-in-time correctness
  Features must be computed by event time, not current time
  Offline store must support training only with features that are available during prediction
  Offline store should support paritioning features by time

Training-serving skew
  When new features are added, run backfill jobs offline to fill offline store
  Re-use spark jobs for streaming feature transformation (for online feature store) and backfill (for offline feature store)
