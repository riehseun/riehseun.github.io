Distributed Training


# Requirement and Problem

How many models are there? Over 100
What kind of models? LLM, Feed recommendation, harmful content detection, product search, fraud detection
What is the size of training data? XBs
Should be model be continuously trained? Yes
How many GPUs are available? 5k
How much time can each training take? Less than a week

ML Problem
  Large compute cluster perform parallel execution

# High-Level Design

Data layer -> Compute layer -> Orchestration layer -> Training runtime layer -> Reliability layer -> Observability layer

# Detailed Design

Data layer
  Use distributed object storage
  Shard data for parallel reading, shard by sample_id for data parallelism
  Co-host data and compute to reduce cross-rack traffic

Compute layer
  Create a node with 8 GPUs
  Use Kubernetes to manage cluster for auto-scaling

Training runtime layer
  Data parallelism - each node get a shard
  Model parallelism - split layers across GPUs
  Pipeline parallelism - split model by layers

Reliability
  Checkpointing - if node is down, restart from last checkpoint

Observability
  GPU utilization
