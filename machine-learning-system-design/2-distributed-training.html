# Requirement

How many models are there? Over 100
What kind of models? LLM, Feed recommendation, harmful content detection, product search, fraud detection
What is the size of training data? XBs
Should be model be continuously trained? Yes
How many GPUs are available? 5k
How much time can each training take? Less than a week

# Design

Data ingestion -> Data processing -> Raw storage -> Offline feature store -> Compute -> Orchestration -> Training -> Model registry

Data ingestion
  Both real-time and batch data
  Ex. Kafka

Data processing
  Real-time processing 
  Ex. Spark structured streaming 

Raw storage 
  Append only (immutable)
  Ex. S3
  
Offline feature store (curated data)
  Extract features into feature store
  Shard data for parallel reading, shard by sample_id for data parallelism
  Store in parquet format for fast feature read
  Co-host data and compute to reduce cross-rack traffic
  Compute reads features from feature store when models are trained 
  Ex. Feast

Compute
  Create a node with 8 GPUs
  Use Kubernetes to manage cluster for auto-scaling

Orchestration
  Invokes different model training
  Ex. Kubeflow, Metaflow

Training
  Data parallelism - each node get a shard, gradients are computed locally and syncs via AllReduce
  Model parallelism - split layers across GPUs
  Pipeline parallelism - split model by layers
  Async gradient update over sync for faster training

Model registry
  Track experiments
  Ex. MLflow
  Store model versions and parameters

Re-training
  Trigger when batch of data becomes available or when data distribution drift is detected

# Discussion
