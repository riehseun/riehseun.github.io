Product search


# Requirement and Problem

What do users search? Products in commercial website, like Amazon
What does a product consist of? Text description, image, video, reviews
Should the system support multiple languages? No, English only
Does each product have category? Yes
Can each product have more than one category? Yes
Should search result be personalized to user? Yes
Should search allow filters? Yes
Should search result be allowed to be sorted? Yes
How often is the product updated? Once every 6 months on average

Is there training data available? No, user-product (under each search) interaction data only
How fast should the system show search results? Under 200ms
How many products are there in system? 1B

ML problem
  Input - search text
  Output - ranked list of product
  Supervised ranking
Business problem
  Maximize user engagement, thus increase number of total purchases

# Data and Feature Engineering

Training data
  Positive - user bought the product and did not leave negative review, user clicked product, user spend more than 10 seconds viewing the product 
  Negative - users left negative review, user did not click product
  Each data point is associated with query

Features
  Query - query length
  Product - reviews, price, text description, number of comments, comments, number of views, image, video, number of purchase, total $ amount of purchase
  User - gender, age

Feature engineering
  Numerical (number of comments, number of purchase, age, price, review) - standardize (Mean 0, Var 1)
  Categorical dense (gender) - one-hot or embedding
  Categorical sparse (user_id, product_id) - embedding to learn
  Text (search term, comment, description) - BERT
  Image/video - CLIP

# Model Development

Baseline
  Logistic regression to determine, given query, if product is relevant to to user

Architecture
  Candidate generation
    Find similarity between query embedding and product embedding
    Use ANN to generate 10k candidates
  Ranking
    Learn-to-rank
    Find similarity between embedding of user and product
  Re-ranking
    Diversity
    Freshness

Training
  Ranking 
    Loss - binary classification to determine if user finds the product relevant (cross-entropy or log-loss)

# Evaluation

Offline
  Candidate generation - recall@k
  Ranking - NDCG

Online
  Revenue increase

# Detailed Design

Query understanding
  Spell correction
  Query expansion
  Query embedding

Cache
  Store search result in cache (for each age and gender group) and return it

Candidate generation
  Use batch features from feature store
    Pre-compute features for possible queries
    Use product features that don't change often (text description, image, video)
  Compute this offline

Ranking
  Use batch features from feature store
    User features don't change often
  Process dynamic features on the fly (product - number of views, reviews, etc)
  
Re-ranking
  Diversity - penalize products from same vendor
  Freshness - penalize old product
  Business rules
