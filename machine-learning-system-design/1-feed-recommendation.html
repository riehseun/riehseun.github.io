Requirement and problem

  • Business
    • What is the purpose of system? Display a list of videos to user
    • Should videos be personalized for each user? Yes - #1
    • Should the system support multiple language? No
  • Data
    • Is labeled data available? No
    • Can user interaction data be used for training? Yes - #2
  • Constraint
    • How many users are there? 1B
    • How many daily active users are there? 100M
    • How many videos are there? 10B
    • Should model be continously trained? Yes - #3 (if time)
    • How fast should the inference be? Under 200ms - #4 (if time)
  • Problem - ranking problem
  • Input - a user
  • Output - ranked list of video by probability of user clicking it

Data and feature engineering

  • Data
    • User
    • Video
    • User-video interaction
  • Feature
    • User geo location - people in different language and culture are interested in different videos
    • User age/gender/ethnicity - people under different groups... (Can't be used if bias/fairness is problem)
    • Video duration - length of videos are related to video quality
    • Video number of likes/shares/comments - popular videos might be better in quality
    • Whether current user liked/shared/commented video - if current user directly interacted with video, likely that user is very interested in it
  • Engineering
    • Numerical such as geo location, duration, number of likes, etc - standardize (scale and normalize)
    • Categorical (low-cardinality) such as age, gender - one-hot encoding
    • Categorical (high-cardinality) such as ethnicity - learnable embedding
    • Text such as comments - BERT embedding
    • Video such as video itself - CLIP video encoder embedding

Model development

  • Baseline
    • Heuristics - show vidoes by number of total likes
    • Logistic regression to determine if user will click a video or not (Binary classification)
  • Model
    • Ensembles
      • Classification to determine whether user will click video or not
      • Difficult to deal with text, video feature
    • Matrix factorization
      • Learn both user and video embedding based on historical user-video interaction
      • Does not utilize user and video features, thus performance is low
    • Neural network
      • Learn both user and video embedding
      • Uses both user and video features
      • Constrct cost function such that dot product between user and video embedding is close 1 if there is interaction and 0 otherwise
  • Ground truth
    • Explict interaction - like/comment/share
    • Implicit interaction - watch time greater than x%
  • Data split
    • Training set - 56% of data before date x, use implicit interaction as much as possible
    • Validation set - 24 % of data before date x, use implicit interaction as much as possible
    • Test set - 20% of data after date x, use direct interaction as much as possible
  • Data imbalance
    • Undersample negative examples

Inference and evaluation

  • Sampling
    • Out of 100 videos, sample from the top in the ranking
    • Skip similar video (videos are similar if their embedding distance is lower than some value)
    • Skip videos that were previously recommended to user (Give penality to videos if recommended previous to current user)
  • Offline metric
    • Candidate generation - recall
    • Ranking - recall
    • Re-ranking - precision
    • Overall - NDCG
  • Online metric
    • Number of clicks over number of impressions
    • Total video watch time
