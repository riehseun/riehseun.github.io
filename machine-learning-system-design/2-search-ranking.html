Search Ranking


# Requirement

What do users search for? Videos 
Should search result be personalized for each user? Yes
Can video include text description, thumbnail image, likes, shares, comments, tags? Yes
Should the system support multiple languages? Yes
Should the system focus on user engagement or content quality? Quality
Should the system focus on exploitation or exploration? Balance of both
Should the system moderate contents? Yes

How fast should inference be? Under 1s
How many videos are there? 1B
How many users are there? 100M
How many searches does each user make per day? 5, (5 * 100M) / (24 * 3600) QPS
Can peak QPS be 20-30 times the average? Yes
Should model be continuously trained? Yes
Should model handle positional bias? Yes

# Problem

ML
  Input - user and search term
  Output - ranked list of videos
  Supervised ranking

Business
  Maximize user engagement, subject to certain precision (we don't want to just return click-baits)

# Serving

Query -> Query processing -> Cache -> Candidate generation -> Ranking -> Re-ranking -> Output & feedback

Query processing
  Correct spelling mistakes
  Add contexts
  Read query embedding from cache or compute embedding and store it in cache (Ex. FAISS)

Cache
  Return cached search results matching query embedding

Candidate generation ~ 200ms
  Come up with ~1k candidates using ANN between query embedding and video embeddings
  Execute text search from inverted index storing video's textual data, to find relevant videos 
  Gather hundreds of popular and trending vidoes based on speed of increasing positive user engagement
  Gather tens of previously watched videos by the user
  Combine all these to generate candidates

Ranking ~ 400ms
  Use a model that can rank thousands of videos

Re-ranking ~ 100ms
  Use a model that can rank 20-30 videos

# Data

Positive - if user likes/shares videos, leaves positive comments, watches more than half of videos
Negative - if user dislikes videos, leaves negative comments

# Feature

Video
  Image - CLIP's visual encoder
  Video - split into frames, sample them, and use image encoder
  Text description, tags - distlimBERT embedding (for multi-language)
  Number of likes, shares - normalized
  Comments - sample them, and compute weighted embedding discouted by time

User
  Device, time of day, day of week, month of year - bucketize and one-hot
  Country, city, language - BERT embedding

Temporal
  Previous queries that user executed - sample and compute weighted embedding discouted by time

# Model

Candidate generation - two-tower NN
  Video tower concatenates all video features and encode them into one embedding
  User-query tower concatenates all user features and query embedding, and generates one user-query embedding
  Dot product between user-query and video embedding is taken, where 1 means user found search result relevant, 0 otherwise
  Loss is weighted sum of individual losses where each loss is regarding likes, shares, comments, score from match between query and text description (exact match should get very high weight)

Ranking - DeepFM
  Point-wise learn to rank

Re-ranking - cross-encoder
  Ranking using transformer encoder is feasible at inference time only when data points are about 20-30 
  Input is again pair of user-query and video embedding
  Diversity - penalize videos with similar embeddings
  Freshness - penalize old videos

# Discussion

Availability
  Replicate each service in different geo region (trade-off with cost)
Reliability
  Partition data into shards, create replicas for each shared, spread replicas across different geo region
Scalability
  Horizontally scale each service based on traffic pattern
Performance
  Quantization - use fewer bits to represent number
  Pruning - regularization such as drop-out to make network more sparse
Monitoring
  System - QPS, latency
  Model - feature drift, score drift  

Real-time data ingestion
  Ingestion data into Kafka and process it using Spark Structured Streaming
  Store data into both offline and online feature store
  Use fixed-size window to aggregate features, to ensure point-in-time correctness
  Use same processing logic for both offline and online store
Offline feature store
  Store features in columar format (Ex. parquet)
  For model training
  Ex. S3
Online feature store
  For model inference
  Ex. Redis

Exploration
  Direct 10-15% of recommendation to exploration (Ex. bandit - thompson sampling)

Moderation
  When users upload videos, use a separate model to check if videos are harmful
  After vidoes are uploaded, use another model to check if videos are harmful (features that contributes to harmfulness can change after video gets uploaded, for example, harmful comments)

Bias
  Positional - penalize videos that were ranked high for more than certain duration of time, and apply this during re-ranking
 
Continuous deployment
  A/B testing - direct small portion of traffic to candidate, increase traffic to candidate if it performs better than production model
