# Problem

ML
  Input - user
  Output - list of videos in order of relevance
  Supervised ranking problem

Business
  Maximize user engagement, subject to certain precision (system shouldn't recommend just click baits to increase user engagement)

# Design

User -> Feature store -> Candidate Generation -> Ranking -> Re-ranking -> Output -> Data Lake -> Training

# Data

Positive if user likes/shares video
Positive if user watches more than half of video
Negative if user dislike video
Negative if user does not click recommended video

# Feature

User
  Country
Video
  Title, description - distlimBERT for multi-langugage
  Tag - CBOW since semantic understanding is not important for tag
  Resolution, duration - standardize (mean 0, variance 1)
  Langugage - embedding (categorical)
  Comments - sample and compute averge embedding
  Date uploaded
  Image (thumbnail) - image encoder like CLIP
  Audio - audio encoder
User-Video interaction
  Number of like, dislike, share - standardize
Context
  Device, time of day - bucketized and one-hot
Author
  Popularity of videos (measured by number of likes and total watch time) uploaded by author - standardize
Temporal
  Previous videos watched since opening app or coming to website, previous videos clicked but not watched - compute average embedding

# Model

Baseline
  Heuristics - rank posts by total number of likes or total watch time
  Logistic regression - binary classification to predict whether users will interact with video where like/share/comment/watch can be separate classiction head and total loss is computed by weighted average of individual losses
  
Architecture
  Candidate generation
    Two-tower architecture ()
      User tower computes user embedding
      Video tower with embedding of post_id (video features are not used)
      Take dot product between user and video embedding (1 if close , 0 otherwise)
      Ground truth - 1 if user intected with video, 0 otherwise
    ANN to search videos that are close to user (based on similarity between user and video embeddings)
    Blend the result with popular and trending videos to come up with 10k candidates
  Ranking
    DNN with feature crosses
  Re-ranking
    Diversity - penalized videos from same authors or similar topics and description (meatures by embeddings of topics and description)
    Freshness - penalized old videos
    Deduplication - avoid videos with simialr embeddings
 
Training
  Learn to rank
  Cross-entropy (log-loss) for each classification head - 1 if user will like/share/comment/watch, 0 o/w
  Loss - weighted average of individual losses

# Evaluation

Offline
  Candidate generation
    Recall@k to measure percentage of actual relevant videos that model could retrieve
    MRR - not suitable since this metric only cares about the first recommended video in the output list
  Ranking
    Precision@k to measure percentage of relevant videos in recommendation list shown to user
    mAP 
  Overall
    NDCG 

Online
  Total watch time of all users in the platform
  Increase in user engagement measured by total number of like/share/comment
  Decrease in number of videos that are clicked but not watched

# Problem

ML - given user as input, output a list of videos ranked by relevance to the user. Supervised ranking problem
Business - maximize user engagement measured by direct and indirect interaction, subject to minimum precision (not to recommend click baits)

# High-level

User -> Cache -> Candidate generation -> Ranking -> Re-ranking -> Output -> Collect feedback & re-train 

Cache - respond using cached recomendation if generation takes longer than 1s
Candidate generation - select 1% videos out of 1B videos. Also, add-in popular and trending videos
Ranking - use a model to select 1% from candidate generation
Re-ranking - apply additional rules such as freshness and diversity

# Data

Positive - if user watches a certain percentage of videos, users likes video, user leaves positive comments about video, user shares video
Negative - if users skip videos, user watches less than certain percentage, user dislike video, user leaves negative comments about video

# Feature

Video
  Video - encode using CLIP
  Thumbnail - encode using CLIP
  Likes, dislike, shares - normalize
  Comments - sample them and compute average embedding
  Tags - word2vec like CBOW (context awareness not needed)
User
  Previous videos that users liked, disliked, shared, left positive comments, left negative comments - sample and compute average embedding, discount by time
  City, country - embedding
  Language - embedding
Context
  Time of day, device type, month - bucketized and one-hot (or just embedding)
Temporal
  Videos that users liked, disliked, shared, left positive comments, left negative comments in current session - sample and compute average embedding, discount by time

# Model

Baseline
  Simply suggest videos ranked by number of likes

Candidate generation - two-tower
  Video tower encodes all features of videos
  User tower encode all user features
  Dot product between video and user encoder is taken, which is closer to 1 if video and user are relevant, 0 otherwise
Ranking - list-wise learn to rank
  More compute intensive than point-wise or pair-wise, but more accurate in terms of computing ranked lists
Re-ranking
  Freshness - penalize videos uploaded in the past
  Diversity - penalize videos that has simialr contents (similar video embedddings)

# Discussion

Content moderation - use a separate model to filter out harmful videos when users upload them
Exploration vs exploitation - direct 10-15% towards exploration and rest to exploitation
Continuouly training - store user direct and indirect feedbak into offline feature store (after processing features), then use them to re-train the model 

Availability
  Use multiple replicas of each service in different geo region (trade-off with cost)
Scalability
  Horizontally scale each services (scale based on traffic pattern)
Reliability
  Parition feature store into shards (Ex. consistent hashing) and spread replicas into data centers across different geo regions
Performance
  Use cached response if latency exceeds certain threshold
  Optimize model - quantization (limit number of bits to represent numbers), pruning, distillation (if teacher model is available)
Monitoring
  System metrics - latency
  Model metrics - feature drift, data drift
