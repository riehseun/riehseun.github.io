# Requirement and Problem

What should chatbot do? Generate response upon user query
What type of query does user input? General query
Should chatbot generate response based on domain specific knowledge? Yes
What domain specific knowledge should chatbot contain? Could be company internal information
Does user query include text, image, video? Text only

Is training data available? No
How fast should the generation be? Under 200ms
Should model be deployed to mobile devices? No
Should model be continuously trained? Yes
What is expected QPS? 100
What is peak QPS? 1000
How often is knowledge updated? New documents are avaiable every month
Should the system support multi-turn conversation? Yes
Should the system personalize response? No

ML problem
  Input - user prompt
  Output - response
  Generation

Business problem
  Provide factual response to users

Data source
  Internal document

  Document cleaning
  Chunking

  Indexing

Training data
  Human can rate response for each query and response pair
  Rating can be used to fine-tune the model

# Model Development

Architecture
  Start with pre-trained model
  Fine-tune with training data
  RAG
    Query understanding
      Correct spelling mistakes
      Add context to given query
      Compute query embedding
    Retriever
      Use ANN to find top-k chunks
    Re-ranker
      Order top-k chucks based on relevance to query
    
# Evaluation

Offline
  Retriever - recall 
  Reranker - precision
  Model - factual accuracy, number of harmful response

Online
  A/B testing
