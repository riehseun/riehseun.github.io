Model selection

  • Linear or logistic regression
    • Pros
      • Simple to implement
      • Explainable
    • Cons
      • Low accuracy
  • Decision trees or XGBoost
    • Pros
      • No need to handle missing values
      • No need to scale features
      • Explainable
    • Cons
      • New data or features require entire model to be retrained
  • Neural network
    • Pros
      • Handles unstructured data
      • Can produce embeddings
    • Cons
      • Lots of dataprep
      • Lots of hyperparameter tuning
      • Not explainable

  • Classification

    • ROC-AUC
      • Used for binary classification
      • Does not penalize how far off predicted score is from the actual label
      • Insensitive to well-calibrated probabilities
    • PR-AUC
    • Confusion matrix
  • Regression
    • MSE
    • MAE
    • RMSE
  • Ranking
    • Precision@k
    • Recall@k
    • MRR
    • mAP
      • Assume total items are 10. Recommended items are 5 and user find 3 of them relevant
      • \( AP@N = \frac{1}{m} \sum_{k=1}^N P(k) \cdot rel(k) = \frac{1}{10} \cdot (1*\frac{1}{1} + 1*\frac{2}{2} + 0*\frac{2}{3} + 1*\frac{3}{4} + 0*\frac{3}{5}) = 0.275 \)
    • mAR
      • \( AR@N = \frac{1}{m} \sum_{k=1}^N R(k) \cdot rel(k) = \frac{1}{10} \cdot (1*\frac{1}{10} + 1*\frac{2}{10} + 0*\frac{2}{10} + 1*\frac{3}{10} + 0*\frac{3}{10}) = 0.06 \)
    • nDCG
      • Assume the documents are \( D1, D2, D3, D4 \) in the order of relevance
      • Assume human rates the documents on scale of 0-3 (3 is highly relevant, 0 is merely relevant) such that
        • \( D1=3, D2=2, D3=3, D4=0 \)
      • Cumulative gain simply adds
        • \( 3 + 2 + 3 + 0 = 8 \)
      • Discounted cumulative gain (DCG) penalizes if highly relevant document appears lower in the result
        • \( \frac{3}{log(1+1)} + \frac{2}{log(2+1)} + \frac{3}{log(3+1)} + \frac{0}{log(4+1)} = 3 + 1.262 + 1.5 + 0 = 5.762 \)
      • Discounted cumulative gain for a query with long result will be higher due to its length rather than its quality
      • Normalized discounted cumulative gain (NDCG) is computed by (DCG / IDCG) where IDCG is DCG of ideal ordering
        • Near 1 means good result. Near 0 means bad result
        • Caveat - NDCG does not penalize irrelevant search result

  • RAG
    • Retrieval
      • Context precision
        • \( \dfrac{\text{number of relevant chuck in top k}}{\text{number of retrieved chuck = k}} \)
        • For each query, determine if chunks returned by retriever is relevant or not
        • Compute precision @ k for each k in the retrieved chunks
          • \( \text{precision @ k} = \dfrac{\text{true positive @ k}}{\text{true positive @ k} + \text{false positive @ k}} \)
        • Compute context precision @ K by averaging precision @ k for all relevant chunk in top k results
          • \( \text{context precision @ K} = \dfrac{\displaystyle\sum_{k=1}^{K} \text{precision @ k} \times v_{k}}{\text{total number of relevant chunks in top k result}} \)
          • \( v_{k} \) is an indicator whether chunk is relevant or not
          • \( K \) is total number of chunks
      • Context recall
        • \( \dfrac{\text{number of relevant chuck in top k}}{\text{total number of relevant chuck}} \)
    • Generation
      • Faithfulness
        • \( \dfrac{\text{total number of claims}}{\text{number of clains that can be inferred from given context}} \)
      • Answer relevance
        • Calculate mean cosine similarity between original question and generated (artificial) questions that are reverse engineered based on provided answers
        • \( \dfrac{1}{N}\displaystyle\sum_{i=1}^{N} \cos(E_{g_{i}}, E_{o}) = \dfrac{1}{N}\displaystyle\sum_{i=1}^{N} \dfrac{E_{g_{i}} \cdot E_{o}}{\|E_{g_{i}}\|\|{E_{o}}\|} \)
        • \( E_{g_{i}} \) is embedding of generated question \( i \)
        • \( E_{o} \) is embedding of original question
        • \( N \) is number of generated question (typically 3)
    • End-to-end
      • Summarization score
        • \( \text{QA score} = \dfrac{\text{number of correctly answered questions}}{\text{total number of questions}} \)
        • \( \text{consciousness score} = \dfrac{\text{length of summary}}{\text{length of context}} \)
        • \( \text{summarization score} = \dfrac{\text{QA score+consciousness score}}{\text{2}} \)
      • Context entities recall
        • \( \dfrac{|CE \cap GE|}{|GE|} \)
        • \( CE \) - entities present in provided answer
        • \( GE \) - entities present in ground truth
        • \( |CE \cap GE| \) - entities that are both in provided answer and ground truth
