Machine Learning


1. What is Machine Learning?

Machine learning is a field of AI where models learn patterns from data to make predictions or decisions without being explicitly programmed. Instead of writing rules, we train algorithms on historical data so they can generalize to new data.

2. Supervised vs Unsupervised Learning

Supervised learning

The model learns from labeled data where inputs and outputs are known. The goal is to learn a mapping from features to targets.

Examples:

regression

classification

Unsupervised learning

The model learns patterns from unlabeled data, such as clusters or latent structure.

Examples:

clustering

dimensionality reduction

3. Bias vs Variance

Bias

Error caused by overly simple assumptions in the model.

Variance

Error caused by sensitivity to small changes in training data.

Tradeoff:

Increasing model complexity reduces bias but increases variance.

4. Overfitting vs Underfitting

Overfitting

Model learns noise in training data and performs poorly on new data.

Underfitting

Model is too simple to capture underlying patterns.

Solution methods:

regularization

more data

cross-validation

simpler models

5. What is Cross Validation?

Cross validation evaluates model performance by splitting data into multiple train and validation sets. In k-fold cross validation, the dataset is split into k folds and each fold is used once as validation while the rest are used for training.

6. What is Regularization?

Regularization adds a penalty to the loss function to prevent overfitting by discouraging overly complex models.

Common types:

L1 (Lasso) → encourages sparsity

L2 (Ridge) → shrinks coefficients

7. L1 vs L2 Regularization

L1

drives some coefficients to zero

useful for feature selection

L2

shrinks coefficients but rarely zero

stabilizes models with correlated features

8. Gradient Descent

Gradient descent is an optimization algorithm used to minimize a loss function by iteratively moving parameters in the direction of the negative gradient.

Types:

batch

stochastic

mini-batch

9. Why Feature Scaling Matters

Feature scaling ensures features are on similar ranges so optimization algorithms like gradient descent converge faster and distance-based algorithms like KNN work correctly.

Common methods:

standardization

normalization

10. Logistic Regression

Logistic regression is a classification algorithm that models the probability of a class using the logistic (sigmoid) function applied to a linear combination of input features.

Output:

probability between 0 and 1

11. Why Log Loss Instead of MSE in Logistic Regression

Log loss corresponds to maximum likelihood estimation for Bernoulli-distributed outcomes and results in a convex optimization problem, whereas MSE can produce non-convex optimization and poor probability estimates.

12. Precision vs Recall

Precision

Fraction of predicted positives that are actually positive.

Recall

Fraction of actual positives correctly detected.

13. When to Use Precision vs Recall

Precision critical

spam detection

Recall critical

disease detection

fraud detection

14. ROC-AUC vs PR-AUC

ROC-AUC

Measures tradeoff between true positive rate and false positive rate.

PR-AUC

Measures precision vs recall and is more informative when datasets are highly imbalanced.

15. Confusion Matrix

A confusion matrix summarizes classification results by counting true positives, false positives, true negatives, and false negatives.

16. Decision Trees

Decision trees split data based on feature thresholds to minimize impurity measures like Gini impurity or entropy.

Advantages:

interpretable

handles non-linear relationships

17. Random Forest

Random forest is an ensemble of decision trees trained on random subsets of data and features. Predictions are aggregated across trees to reduce variance and improve generalization.

18. Gradient Boosting

Gradient boosting builds models sequentially where each new model corrects the errors of the previous one.

Popular implementations:

XGBoost

LightGBM

CatBoost

19. K-Nearest Neighbors

KNN predicts outputs based on the labels of the K closest data points using a distance metric such as Euclidean distance.

Characteristics:

non-parametric

lazy learner

sensitive to feature scaling

20. Curse of Dimensionality

As dimensionality increases, data becomes sparse and distance metrics become less meaningful, making many machine learning algorithms less effective.

21. Principal Component Analysis (PCA)

PCA is a dimensionality reduction technique that transforms data into orthogonal components that maximize variance.

Benefits:

reduce dimensionality

remove correlated features

22. t-SNE

t-SNE is a nonlinear dimensionality reduction technique used mainly for visualization by preserving local structure of high-dimensional data.

23. K-Means Clustering

K-means partitions data into k clusters by minimizing the within-cluster sum of squared distances between points and cluster centroids.

Algorithm:

initialize centroids

assign points

update centroids

repeat until convergence

24. Evaluation Metrics for Regression

Common metrics:

MSE – penalizes large errors

MAE – robust to outliers

R² – proportion of variance explained

25. Central Limit Theorem

The central limit theorem states that the distribution of the sample mean approaches a normal distribution as sample size increases, regardless of the population distribution.


1. What is Data Leakage?

Data leakage occurs when information from the test set or future data unintentionally influences the training process, causing overly optimistic model performance.

Common causes:

using future data

feature engineering done before train/test split

target leakage

Example:
Predicting loan default using “loan approved” as a feature.

2. How Would You Handle Imbalanced Data?

Common strategies:

Data-level

oversampling minority class

undersampling majority class

synthetic data (e.g. SMOTE)

Algorithm-level

class weighting

cost-sensitive learning

Evaluation

use PR-AUC or F1, not accuracy.

3. Why Is Accuracy a Bad Metric for Imbalanced Data?

Example:

Dataset

99% negative

1% positive

A model predicting all negatives gets 99% accuracy but zero usefulness.

Better metrics:

precision

recall

F1

PR-AUC

4. What Is the Bias–Variance Tradeoff?

Total prediction error consists of:

Bias²

Variance

Irreducible noise

Simple models → high bias, low variance
Complex models → low bias, high variance

Goal: find balance.

5. Why Does KNN Perform Poorly in High Dimensions?

Because of the curse of dimensionality:

distances between points become similar

nearest neighbors are less meaningful

data becomes sparse

6. Why Use Cross Validation Instead of a Single Train/Test Split?

Cross validation:

reduces variance of evaluation

uses data more efficiently

provides more stable performance estimates

Most common:

k-fold cross validation

7. What Is Feature Engineering?

Feature engineering is the process of transforming raw data into meaningful input features that improve model performance.

Examples:

log transformations

interaction terms

aggregation features

encoding categorical variables

Often more important than model choice.

8. What Is Multicollinearity?

Multicollinearity occurs when two or more features are highly correlated.

Problems:

unstable regression coefficients

inflated variance

Solutions:

remove correlated features

use PCA

use regularization (L2)

9. Difference Between Generative and Discriminative Models

Generative models

Model joint probability:

P(X, Y)

Examples:

Naive Bayes

Hidden Markov Model

Discriminative models

Model conditional probability:

P(Y | X)

Examples:

Logistic Regression

Support Vector Machine

10. What Is Model Calibration?

Calibration measures whether predicted probabilities reflect true probabilities.

Example:

If model predicts 0.8 probability, event should occur ~80% of the time.

Calibration methods:

Platt scaling

isotonic regression

11. Why Do We Use Log Loss?

Advantages:

penalizes confident wrong predictions heavily

corresponds to maximum likelihood estimation

produces well-calibrated probabilities

12. Offline vs Online Evaluation

Offline evaluation

historical dataset

cross-validation

fast iteration

Online evaluation

real users

A/B testing

Example: experiments run at Netflix or Amazon.

13. What Is A/B Testing?

A/B testing compares two model versions by randomly assigning users to different variants and measuring performance differences.

Key metrics:

conversion rate

engagement

revenue

14. What Happens if Training and Test Distributions Differ?

This is called:

Distribution shift

Types:

covariate shift

label shift

concept drift

Solutions:

retraining

domain adaptation

monitoring data drift

15. What Is Concept Drift?

Concept drift occurs when the relationship between input features and the target changes over time.

Example:

Fraud detection patterns changing as fraudsters adapt.

Solutions:

periodic retraining

online learning

drift detection

The 3 “Meta Questions” ML Interviewers Love

Many interviews end with questions like:

1️⃣ “Your model accuracy dropped in production. What do you check?”

Expected steps:

training/serving mismatch

2️⃣ “Model performs great offline but poorly in production”

Possible causes:

data leakage

evaluation metric mismatch

distribution shift

3️⃣ “You have limited labeled data”

Solutions:

semi-supervised learning

active learning

transfer learning

weak supervision  


The Question

Your model has 95% accuracy.
Is it good?

Most candidates say yes — and fail the interview. ❌

The Correct Thought Process

Accuracy alone tells almost nothing without context.

You must ask:

1️⃣ Is the dataset balanced?

Example:

Dataset

95% negative

5% positive

A model predicting all negatives gets:

Accuracy = 95%

But the model is useless.

This is common in:

fraud detection

disease detection

spam detection

The Correct Metrics

Instead of accuracy, evaluate:

Precision

When the model predicts positive, how often is it correct?

Recall

How many actual positives are detected?

F1 score

Harmonic mean of precision and recall.

PR-AUC

Best metric for imbalanced datasets.

Example

Fraud detection dataset:

Metric  Value
Accuracy  95%
Recall  10%

Meaning:

The model detects only 10% of fraud cases.

So the model is very poor, despite high accuracy.

The Interview Answer

A strong candidate says something like:

Accuracy alone is not enough to evaluate model performance. If the dataset is imbalanced, a model can achieve high accuracy by predicting the majority class. I would examine metrics such as precision, recall, F1 score, or PR-AUC to better understand performance, especially for rare events like fraud or disease detection.

Follow-Up Question Interviewers Often Ask

They may ask:

Your model has high recall but low precision. What does that mean?

Answer:

The model detects most positive cases but also generates many false positives.

Example:

good for disease screening

bad for spam filtering

Another Trick Question

Interviewers may ask:

Your offline metrics improved, but production performance dropped. Why?

Possible reasons:

1️⃣ Data leakage
2️⃣ Distribution shift
3️⃣ Training-serving skew
4️⃣ Incorrect evaluation metric
5️⃣ User behavior changed

What This Question Tests

It checks if you understand:

class imbalance

evaluation metrics

real-world ML problems

These skills matter more than memorizing algorithms.
