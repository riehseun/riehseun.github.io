Problem

ML problem

  • Given an AD, compute probability of user click
  • Input
    • An AD
  • Output
    • List of ADs ranked by probability of being clicked

ML category

  • Supervised, ranking problem
    • Content-based filtering
      • Needs to know user interests to match them with the ADs
      • Since no data is available about user interest, this can't be an option
    • Collaborative filtering
      • Needs to know user historical interaction with the ADs
      • Since no data is available about user historical interaction, this can't be an option
    • Learning to rank
      • Pointwise
        • Given a user and an AD, compute probability of click
        • Best option
      • Pariwise
        • Given two ADs, compute which AD is more probable to be clicked
        • Compute expensive
      • Listwise
        • Given a list of ADs, compute the optimal ordering
        • Compute expensive

Model development

Model selection

  • Logistic regression
    • Fast training and inference, interpretable
    • Cannot handle complex (non-linear) interaction between features
  • Gradient boosting
    • No scaling needed, handles missing values, interpretable
    • Cannot train embedding layer, not suitable for continual learning
  • Neural network
    • Single NN
      • All features are used as inputs and NN outputs click probability
    • Two tower NN
      • Separate encoder for user and AD features
      • Similarity of user embeddings and AD embeddings is computed

Model training

  • Loss function
    • Constrastive loss function
      • Dot product or cosine similarity is used to compute distances in embedding space
        • Euclidean distance is not used due to curse of dimensionality
      • Softmax is applied to distances to make them sum to 1
      • Cross-entropy is used to measure how close the probabilities are to the ground truth
  • Hyperparameters
    • Size of embedding vector

Evaluation

Offline

  • Precision@k
    • \( \dfrac{\text{Number of relevant ADs in top k items}}{\text{k}} \)
    • Does not consider ranking quality
  • Recall@k
    • \( \dfrac{\text{Number of relevant ADs in top k items}}{\text{total number of relevant ADs}} \)
    • Does not consider ranking quality
  • Mean reciprobal rank (MRR)
    • \( MRR = \dfrac{1}{m} \displaystyle\sum_{i=1}^{m} \dfrac{1}{\text{rank}_{i}} \)
      • \( {\text{rank}_{i}} \) is position on which the first relevant item appears in the output list
    • Consider only the first relevant AD in each output list, then average the reciprocal rank
    • Ignores other relevant ADs in the list other than the first one
  • Mormalized discounted cumulative gain (nDCG)
    • \( DCG_{p} = \displaystyle\sum_{i=1}^{p} \dfrac{\text{rel}_{i}}{log_{2}(i+1)} \)
      • \( \text{rel}_{i} \) is ground truth relevance score of AD ranked at \( i \)
    • \( nDCG_{p} = \dfrac{DCG_{p}}{IDCG_{p}} \)
      • where \( {IDCG_{p}} \) is \( {DCG_{p}} \) of ideal ranking
    • We only have training data that consist of 0 or 1 representing user click on AD, not the ordering of ADs for each user
  • Cross-entropy
    • Multiply ground truth label (0 or 1) with the probability of AD being clicked for each AD, them sum it to assess the model performance

Online

  • Click-through rate (CTR)
    • \( \dfrac{\text{number of clicked ADs}}{\text{total number of shown ADs}} \)

Serving

Components

  • Indexing pipeline
    • Object storage
      • Stores new images
    • Indexing service
      • Indexes the embeddings of new images so that NN search can easily find it
    • Index table
      • Stores images embeddings in sorted order
  • Prediction pipeline
    • Embedding generation service
      • Preprocess the input images and use trained model to determine its embedding
    • Nearest neighbor service
      • Retrieve similar images from embedding space
      • Option 1. exact NN
        • Time \( \text{O}(ND) \) where \( N \) is total number of points and \( D \) is dimension
        • \( N \) could be billions
      • Option 2. approximate NN
        • Time \( \text{O}(\text{log}(N)D) \)
        • Option 1. tree-based
          • Similar to gradient boosted tree
          • Ex. R-trees, Kd-trees, Annoy (Approximate Nearest Neighbor Oh Yeah)
        • Option 2. locality sensitive hashing (LSH)
          • Use hash function to reduce the dimensions of points and group them into buckets
          • Maps points in close proximity to same bucket
          • LSH searches only those points in the same bucket
        • Option 3. clustering-based
          • Group points based on simiarities
          • Algorithm only searches the subset of points in the cluster
      • Due to algorithm runtime, we choose ANN
    • Re-ranking service
      • Filters inappropriate results, removes duplicates, etc

Workflow

  • New image
    • Images are first stored in object storage
    • Indexing service contacts embedding generation service to get embeddings of images stored in object storage
    • Embeddings are stored in index table for fast search
  • Query image
    • Images are first preprocessed
    • Embedding generation service computes embedding of query image via trained model
    • NN finds similar images between query image and images stored in index table based on their embedding distance
    • Candidate similar images go through re-ranking process
    • Final list of similar images are showned in the order based on similarity score
