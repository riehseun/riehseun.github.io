Model development

Model selection

  • Retrival augmented generation
    • Data ingestion
      • Text splitter
        • Split documents into chunks
        • We want to pass only the relevant texts to LLM
      • Embedding
        • Create embedding for each chunk
        • We want to use embedding to find the most relevant chunks
      • Vector database
        • Stores chunks and embeddings
    • Query
      • System prompt
      • Query expansion
        • Produces standalone questions from chat history
        • Can include spelling correction, adding additional keyword, etc
      • Retriever
        • Executes search on vector database using standalone prompt
        • Outputs sections of the documents that are most relevant to standalone prompt
        • Focused on optimizing recall
      • Re-ranker
        • Takes \( k \) candidate chunks from retriever and reorders them
        • Permutes the chunk to remove noisy parts and only keep most relevant ones
        • Focused on optimizing precision while maintaining recall
      • Generation
        • Original document of most relevant sections plus a system prompt are sent to LLM
        • LLM response includes citations to the referenced documents
