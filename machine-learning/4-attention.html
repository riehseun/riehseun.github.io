Attention

  • Let the decoder decide which encoder outputs are most useful for decoder at current time step.
  • Something like mini NN: decoder's current state and encoder outputs are used as inputs and produces context vector as ouput.
  • During training, we have access to ground truth tokens, which are used input for decoder.
  • During inference, we only have access to input sequence. In this case, input for decoder at each time is just decoder's output token from previous time step.

Encoder-decoder

  • Encoder is usually LSTM or BiLSTM.
  • Pass final state of encoder into decoder.
  • Input tokes for encoder: input sequence.
  • Input tokes for decoder: ground truth tokens.

Teacher forcing

  • At each step making a prediction, peek into the right answer and feed it into the next time step.
  • Input is represented by and target is represented by 1.
  • Input token is fed into input coder, and is transformed to key and value vector.
  • Target token goes into pre-attention decoder, is transformed to query vector Q.
