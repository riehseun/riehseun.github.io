Tokenization

  • Word
    • Large vocabulary size
    • Large risk of OOV words (words seen at inference time but not seen in training)
  • Sub-word
    • Medium vocabulary size
    • Medium risk of OOV words
  • Character
    • No OOV words
    • Very large sequence length, hurts processing time during inference
    • Hard to represent meaning of char

Word representation

  • One-hot encodeing - can't compute similarity between words as distance because all vectors are orthogonal to one another
  • Word2Vec
    • CBOW - predict target words from surrounding workds
    • Skip-gram - predict surrounding workds from target words

RNNs

  • Use all hidden representations of previous tokens to compute the current output
  • Hard to remember hidden representation of word appeared long-time ago in the sequence
  • LSTM
    • Large sequential computation leads to vanishing gradient (multplying lots of small numbers lead to near zero)

Self-attention mechanism

  • Take a query (that is a word in sentence)
  • Compare the query with keys (that are all other words in sentence)
  • And take associated values
  • Computation is done in matrix (which GPUs support)

Transformer architecture

  • Sentence is decomposed into tokens
  • [BOS] token is prepended and [EOS] token is appended
  • Positional embedding is added to embedding of each token (to produce in input)
  • Input is projected into Q,K,V (via W_Q, W_K, W_V, which are all learned by the model)
  • Apply attention formula to product output

Transformer-Based Models & Tricks

Position embeddings

  • Idea - tokens close together are highly related, tokens far away from one-another are less related
  • RoPE (Rotary Position Embedding)
    • Idea - rotate query and key vector by some angle
    • Create a 2 by 2 rotation matirx
    • Multiply vector by the matrix, result results in vector rotated by some angle

Layer normalization

  • PostNorm -> PreNorm -> RSMNorm

Attention approximation

  • Silding window attention - restrict attention to neighboring tokens only to reduce computation, window can be thousands of tokens
  • Sharing attention heads - share projection matrices for key and value (not key), key and value come a lot during decoding!

Transformer-based models

  • Encoder-decoder - T5
  • Encoder - BERT
  • Decoder - GPT

BERT deep dive

  • [CLS] - placeholder at the beginning of sequence
  • [SEP] - aims to separate sentences
  • Multi-stage training
    • Pre-train using MLM (Masked Language Modelling) and NSP (Next Sentence Prediction)
      • MLM - replace input with mask (80% of time), replace input with random word (10% of time), or leave input unchanged (10% of time)
      • NSP - pick two sentences from corpus, they either follow each other (50% of time), or not (50% of time), task is to determine if sentence A follows sentence B
    • Fine-tune for target task
  • Segment encoding - added on top of positional and input embedding, A if first sentence, B if second sentence, learned during training
  • Inference can be slow, cost can be high (Solution - DistillBert, where it removed NSP step and dynamic masking per epoch)
Reference: CME 295 - Transformers & Large Language Models
