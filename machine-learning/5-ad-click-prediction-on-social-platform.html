Clarify requirements

  • Business objective
    • What is the business objective? Maximize revenue
    • Where are the Ad placed such as pop up, timeline, etc? Timeline only
    • Does every Ad generate the same revenue? Yes
    • Can the same Ad be showed to the user more than once? Yes
  • Features that the system supports
    • Can users report or hide the Ads? Yes
    • Can we access user's current location? Yes
    • What kinds of features are available for an event? description, price, location, date and time, etc
  • Data
    • Is annotated datset available? No, data is from user-ad interaction only
    • Can we assume impression without clicks, hide, report as negative examples? Yes
  • Constraints
    • Should the model be trained continuously? Yes
  • Scale
  • Performance

  • Specify input and output
    • Input - user
    • Output - a list of Ads sorted by likelihood of being clicked

Model development

  • Model selection
    • Logistic regression
      • Pros
        • Fast inference
        • Fast training
        • Interpretable
      • Cons
        • Non-linear problems can't be solved
        • Cannot handle dependency between features
    • Feature crossing + LR
      • Create new feature by combining existing features
      • This way, non-linear relationship can be captured
      • Pros
        • Capture feature interations better
      • Cons
        • Manual process to choose features to be crossed, which also requires domain knowledge
        • Cannot still capture complex interactions
        • When original features are sparse, cardinality of crossed features become very sparse
    • GBDT
      • Pros
        • Interpretable
      • Cons
        • Not suitable for continual learning
        • Cannot train embedding layers
    • GBDT + LR
      • Use GBDT just to select features and also extract features
      • Pros
        • Extracted features are more predictive
      • Cons
        • Not suitable for continual learning
        • Cannot capture complex interactions
    • Neural network
      • Single NN - use original features as input and NN outputs the click probability
      • Two-tower NN - user encoder and ad encoder are used. Similarity between user and ad embeddings is used to determine relevance
      • Pros
      • Cons
        • Most features are filled with zero
    • Deep & cross network (DCN)
      • Deep network - use DNN to learn complex features
      • Cross network - automatically captures feature interactions and learns good feature crosses
      • Outputs of deep and cross network are concatenated to make final predictions
      • Pros
        • More effective than NN because it implicitylu learns feature crosses
      • Cons
        • Cross network only models certain feature interactions
    • Factorization machines (FM)
      • Learn embedding vector for each feature to find all pairwise feature interactions
      • \( \hat{y}(x) = w_{0} + \displaystyle\sum_{i} w_{i}x_{i} + \displaystyle\sum_{i}\displaystyle\sum_{j} <v_{i}, v_{j}> x_{i}x_{j} \)
        • \( x_{i} \) - ith feature
        • \( w_{i} \) - learned weight
        • \( v_{i} \) - embedding of ith feature
        • \( <v_{i}, v_{j}> \) - dot product between two embeddings
      • Pros
        • Captures pairwise feature interactions
      • Cons
        • Cannot learn sophisticated higher-order feature interactions
    • Deep factorization machines (DeepFM)
      • DNN capture sophisticated higher-order features
      • FM captures low-level pairwise feature interactions
      • Pros
      • Cons
  • Model training
    • Construct dataset
      • If user clicks the Ad in less than \( t \) seconds after Ad is shown, positive. Otherwise, negative
    • Choose loss function
      • Binary classification, thus use cross-entropy

Evaluation

  • Offline
    • Cross-entropy
      • Ground truth
        • Ad1 - 1 (clicked)
        • Ad2 - 0 (not clicked)
        • Ad3 - 1 (clicked)
      • Model A
        • Ad1 - p(click) = 0.8
        • Ad2 - p(click) = 0.3
        • Ad3 - p(click) = 0.95
      • Model B
        • Ad1 - p(click) = 0.2
        • Ad2 - p(click) = 0.6
        • Ad3 - p(click) = 0.6
      • Cross-entropy
        • \( \text{CE} = -\displaystyle\sum_{i} y_{i}log\hat{y_{i}} + (1-y_{i})log(1-\hat{y_{i}}) \)
        • \( \text{CEA} = -(1log0.8 + (1-0)log(1-0.3) +1log0.95) = 0.273 \)
        • \( \text{CEA} = -(1log0.2 + (1-0)log(1-0.6) +1log0.6) = 1.319 \)
    • Normalized cross-entropy
      • Considers only the first item, thus not good
  • Online
    • Click-through rate = total number of clicked Ads / total number of shown Ads
    • Conversion rate = total number of conversions / total number of impressions
    • Hide rate
    • Revenue life

Deployment and serving

  • Online learning pipeline
    • Continuously train new models with new data, evaluate trained model, and deploy them
  • Prediction pipeline
    • Event filtering
      • Narrow down 1M events to a small subset
      • Simple rules can be used
    • Ranking service
      • Computes features for each <user, event> pair
      • Sorts events based on probabilities computed by the model
      • Ouputs top k most relevent events
