Frame problem as ML task

  • Define ML objective
    • Predict which listing the user will click next, given the listing that user is currently viewing
  • Specify input and output
    • Input - listing that user is currently viewing
    • Output - a list of listings ranked by probability of being clicked
  • Choose right ML category
    • Session-based recommendation system
      • Map each listing into embedding vector
      • Embeddings are close when the two listings frequently co-occur in user's browsing history

Model development

  • Model selection
    • Neural networking to learn embeddings
  • Model training
    • Construct dataset
      • Negative sampling
        • Move sliding window through search sessions
        • As the sliding window moves, use central listing and its nearby listings as positive pairs
        • As the sliding window moves, use central listing and randomly sampled listings as negative pairs
        • Positive pairs have ground truth label 1 and negative pairs have ground truth label 0
    • Choose loss function
      • \( \displaystyle\sum_{(c,p) \in D_{p}} \text{log} \dfrac{1}{1+e^{-E_{p}E_{c}}} + \displaystyle\sum_{(c,n) \in D_{n}} \text{log} \dfrac{1}{1+e^{E_{p}E_{c}}} + \displaystyle\sum_{(c,b) \in D_{booked}} \text{log} \dfrac{1}{1+e^{-E_{b}E_{c}}} + \displaystyle\sum_{(c,n) \in D_{hard}} \text{log} \dfrac{1}{1+e^{E_{n}E_{c}}} \)
      • First term computes loss over positive pairs
      • Second term computes loss over negative pairs
      • Third term helps push embeddings of central listing closer to embeddings of eventually booked listings
      • Third term helps push embeddings of central listing away from embeddings of random listings
