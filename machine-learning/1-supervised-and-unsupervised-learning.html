## Explain evaluation used for classification

Precision - how often model is correct when predicting target class, true positive / (true positive + false positive)
Recall - whether model can find target class, true positive / (true positive + false negative)
Accuracy - percentage of predictions that were correct, (True Positive + True Negative) / (True Negative + True Positive + False Negative + False Positive)
F1 - average of precision and recall

## Explain confusion matrix

Table showing predicted versus actual (true positive, false negative, false positive, true negative)

## Explain difference between PR-AUC and ROC-AUC

ROC-AUC - true positive vs false positive, area beneath the curve (If 0.5, random guess, If 1, perfect model)
PR-AUC - precision vs recall, area beneath the curve

## Explain when to use precision vs recall

Precision - when missing target class is less critical than incorrectly labeling target class (Ex. spam detection)
Recall - when incorrect lable is less important than missing target class (Ex. fraud detection)
Both - (Ex. cancer detection)

# Explain k-means clustering

Partition data into K clusters
  Elbow method - plot variation (within-cluster sum of squared errors) against number of clsuters, when the curve elbows, adding more cluster does not benefit
  Silhouette score - measures how similar an object is to its own cluster versus other clusters (iterate over different values of k to see which k results in best average score), more computationally expensive than elbow method
Randomly pick centroids of each cluster
Assign points to closest cluster, then update centroid
Repeat until convergence
  Algorithm finished grouping data points into k clusters
  It is when the centroid of last two clusters are very similar

from sklearn.datasets import load_diabetes
from sklearn.model_selection import train_test_split
from torch.utils.data import Dataset, DataLoader

import torch
import torch.nn as nn
import torch.nn.functional as F

# Data

class CustomDataset(Dataset):

    def __init__(self, X_train, y_train):
        self.x = F.normalize(torch.from_numpy(X_train))  # (m, n)
        self.y = torch.from_numpy(y_train) # (m)

    def __len__(self):
        return len(self.x)

    def __getitem__(self, index):
        return self.x[index].float(), self.y[index].float()

dataset = load_diabetes()
X_train, X_test, y_train, y_test = train_test_split(dataset.data, dataset.target, test_size=0.2)
train_data = CustomDataset(X_train, y_train)
test_data = CustomDataset(X_test, y_test)
train_dataloader = DataLoader(train_data, batch_size=64, shuffle=True)
test_dataloader = DataLoader(test_data, batch_size=1, shuffle=True)

# Model
class LinearRegression(nn.Module):

    def __init__(self, num_features):
        super().__init__()
        # X (m,n) => Y (m, 1)
        self.linear = nn.Linear(num_features, 1)

    def forward(self, x):
        x = self.linear(x)
        x = torch.squeeze(x)
        return x

# Train
n = dataset.data.shape[1]
model = LinearRegression(n)
learning_rate = 0.01
criterion = nn.MSELoss()
optimizer = torch.optim.Adam(model.parameters(), lr=learning_rate)

epoch = 1000

print(model.state_dict())

for e in range(epoch):
    for x_batch, y_batch in train_dataloader:

        optimizer.zero_grad()  # Reset grads

        y_hat = model(x_batch)  # (m, 1)

        # print(f"x_batch: {x_batch.shape}")
        # print(f"y_batch: {y_batch.shape}")
        # print(f"y_hat: {y_hat.shape}")
        # print(y_batch)
        # print(y_hat)

        loss = criterion(y_hat, y_batch)  # Compute loss

        loss.backward()  # Compute gradient

        optimizer.step()  # Adjust learning rate

        if (e+1) % 100 == 0:
            print(f'epoch: {e+1}, loss = {loss.item()}')

print(model.state_dict())

# Inference
diff = 0
with torch.no_grad():
    for x_batch, y_batch in test_dataloader:
        y_pred = model(x_batch)  # no need to call model.forward()
        diff += abs(y_pred-y_batch)
    print(diff)

Multi-class Classification (Softmax Regression)

  • Generalization of logistic regression
  • Number of units in the output layer is \( n^{L} = C \) the number of classes
  • If \( C = 2 \), applying softmax reduces to logistic regression

Activation in the output layer

  • In the output layer \( z^{[L]} = w^{[L]}a^{[L-1]} + b^{[l]} \)
  • The activation in the last layer for softmax is \( a^{[L]} = \dfrac{e^{z^{[L]}}}{\sum_{i=1}^{C}t_{i}} \) where \( t = e^{z^{[L]}} \)

Loss function

  • \( L(\hat{y},y) = -\displaystyle\sum_{j=1}^{C} y_{j}log\hat{y}_{j} \)

Gradient descent

  • \( \textbf{dZ} = \textbf{A} - \textbf{y} \)

K-nearest Neighbor

# Define k - pick odd number to avoid ties
# Compute distance between a new point to all other points
# Pick top k points that are closest
# Amongst top k points, count number of points in each category
# Assign new points to top category

import torch

def find_knn_indices(X_train, X_test_point, k):
    # Calculate Euclidean distance: sqrt(sum((train - test_point)^2))
    distances = torch.sum((X_train - X_test_point) ** 2, axis=1).sqrt()
    
    # Sort distances to find the k smallest
    _, indices = torch.sort(distances)
    
    # Return the indices of the k nearest neighbors
    return indices[:k]

# Example usage:
train_data = torch.randn(20, 3) # 20 points, 3 dimensions
query_point = torch.randn(1, 3) # 1 query point
k_neighbors = 3

nearest_indices = find_knn_indices(train_data, query_point, k_neighbors)

print(f"Query point: {query_point}")
print(f"Indices of {k_neighbors} nearest neighbors in train_data: {nearest_indices}")
print(f"The actual neighbor points are:\n{train_data[nearest_indices]}")

K-means

# Pick k
# Randomly pick k points and make them center of k clusters
# Assign each point to one of k clusters based on distance to centroid
# Adjust centroid of each cluster by taking mean of all points within the cluster
# Repeat until cluster assignment no longer changes

import torch
import numpy as np
import matplotlib.pyplot as plt
from kmeans_pytorch import kmeans, kmeans_predict

# --- 1. Setup and Data Generation ---
# Set random seed for reproducibility
np.random.seed(123)
torch.manual_seed(123)

# Set device to GPU if available, otherwise CPU
if torch.cuda.is_available():
    device = torch.device('cuda:0')
else:
    device = torch.device('cpu')
print(f"Running k-means on: {device}")

# Generate some random data points
data_size, dims, num_clusters = 1000, 2, 3
x_numpy = np.random.randn(data_size, dims) / 6
# Add offsets to create distinct clusters
x_numpy[:300] += np.array([-0.5, -0.5])
x_numpy[300:600] += np.array([0.5, 0.5])
x_numpy[600:] += np.array([-0.5, 0.5])

# Convert numpy array to PyTorch tensor and move to the correct device
x = torch.from_numpy(x_numpy).float().to(device)

# --- 2. Perform K-Means Clustering ---
# The kmeans function returns the cluster IDs for each data point and the final cluster centers
cluster_ids_x, cluster_centers = kmeans(
    X=x,
    num_clusters=num_clusters,
    distance='euclidean',
    device=device
)

# --- 3. (Optional) Predict new data points ---
# Generate some new data for prediction
y_numpy = np.random.randn(100, dims) / 6
y = torch.from_numpy(y_numpy).float().to(device)

# Predict cluster IDs for the new data
cluster_ids_y = kmeans_predict(
    y,
    cluster_centers,
    'euclidean',
    device=device
)

print(cluster_ids_y)
