Clarify requirements

  • Business objective
    • Should the system protect user privacy? Yes
    • Should the detect human faces and license plates? Yes
    • Should the system handle bias w.r.t. race, gender, age, etc? No
  • Features that the system supports
    • Can users report images that are not correctly blurred? Yes
  • Data
    • Do we have annotated dataset? There are 1 million images with human faces and license plates manually annotated
  • Constraints
  • Scale
  • Performance
    • Should the system have low latency? No, images can be processed offline
Reference: Machine Learning System Design Interview, Ali Aminian & Alex Xu

Frame problem as ML task

  • Define ML objective
    • Accurately detect objects in images
    • Accurately identify objects of interest
  • Specify input and output
    • Input - images with zero or more onjects in it
    • Output - location of objects of interest
  • Choose right ML category
    • Object detection is regression problem
    • Object classification is classification problem
    • Two stage networks
      • Region proposal network (RPN) - proposes candidate regions that are likely objects
      • Classifier - classifies each proposed region into an object class
      • Usually slower but accurate
      • Ex. R-CNN, Fast R-CNN, Faster R-CNN
    • One stage network
      • Bounding boxes and object classes are generated simultaneously
      • Ex. YOLO, SDD
Reference: Machine Learning System Design Interview, Ali Aminian & Alex Xu

Data preparation

  • Data engineering
    • Annotated dataset
      • Image path
      • Objects - human face, license plate
      • Bounding boxes - x coordinate, y coordinate, width, height
    • Street view images
      • Image path
  • Feature engineering
    • Resize
    • Normalize
    • Data augmentation
      • Random crop
      • Random saturation
      • Vertical/horizontal flip
      • Rotation/translation
      • Affline transformations
      • Changing brightness, saturation, contrast
      • When rotating/flipping images, the ground truth bounding boxes must also be transformed
      • Offline
        • Augment image before training
        • Training is faster but requires additional storage
      • Online
        • Augment image on the fly during training
        • Training is slower but requires no additional storage

Model development

  • Model selection
    • Two stages network
      • Convolutional layer
      • Region proposal network (RPN)
      • Classifier
  • Model training
    • Regression loss
      • \( L_{reg} = \dfrac{1}{M}\displaystyle\sum_{i=1}^{M} \left[ (x_{i}-\hat{x}_{i})^{2} + (y_{i}-\hat{y}_{i})^{2} + (w_{i}-\hat{w}_{i})^{2} + (h_{i}-\hat{h}_{i})^{2} \right] \)
      • \( M \) - total number of predictions
      • \( x_{i} \) - ground truth top left x coordinate
      • \( \hat{x}_{i} \) - predicted top left x coordinate
      • \( y_{i} \) - ground truth top left y coordinate
      • \( \hat{y}_{i} \) - predicted top left y coordinate
      • \( w_{i} \) - ground truth width
      • \( \hat{w}_{i} \) - predicted width
      • \( h_{i} \) - ground truth height
      • \( \hat{h}_{i} \) - predicted height
    • Classification loss
      • \( L_{cls} = -\dfrac{1}{M} \displaystyle\sum_{i=1}^{M} \displaystyle\sum_{c=1}^{C} y_{c}log\hat{y}_{c} \)
      • \( M \) - total number of detected bounding boxes
      • \( M \) - total number of classes
      • \( y_{i} \) - ground truth label for detection \( i \)
      • \( \hat{y}_{i} \) - predicted class label for detection \( i \)
    • Total loss
      • \( L = L_{reg} + \lambda L_{cls} \)
      • \( \lambda \) - balancing parameter

Evaluation

  • Offline
    • MRR (Mean Reciprobal Rank)
      • \( MRR = \dfrac{1}{m} \displaystyle\sum_{i=1}^{m} \dfrac{1}{rank_{i}} \) where \( {rank_{i}} \) is the position on which the first relevant item appears in the output list
      • Consider only the first relevant item in each output list, then average the reciprocal rank
      • Cons - ignores other relevant items in the list other than the first one
    • Recall@k
      • Number of relevant items in the output list (contaning top k items) / total relevant items in the dataset
      • Cons - denominator (total relevant items in the dataset) can be very big. This negatively impacts recall value
    • Precision@k
      • Number of relevant items in the output list (contaning top k items) / k
      • Cons - does not consider ranking quality
    • mAP (Mean Average Precision)
      • AP
        • \( AP = \dfrac{\displaystyle\sum_{i=1}^{k} precision@i}{n} \) where \( n = \) total relevant items
        • Averages precision@k at different values of k
        • AP is high if more relevant items are located at the top of list
      • mAP
        • Compute AP for each output list, then average them
      • Cons - works well when each item is either relevant or irrelevant. Does not work well for relevance scores
    • nDCG (Normalized Discounted Cumulative Gain)
      • \( DCG_{p} = \displaystyle\sum_{i=1}^{p} \dfrac{rel_{i}}{log_{2}(i+1)} \) where \( rel_{i} \) is ground truth relevance score of image ranked at \( i \)
      • \( nDCG_{p} = \dfrac{DCG_{p}}{IDCG_{p}} \) where \( {IDCG_{p}} \) is \( {DCG_{p}} \) of ideal ranking
      • Example
        • Assume visual search system return images \( I1, I2, I3, I4 \) in the order of relevance
        • Assume human rates the images on scale of 0-3 such that \( I1=3, I2=2, I3=3, I4=0 \) (3 is highly relevant, 0 is merely relevant)
        • \( DCG_{p} = \frac{3}{log(1+1)} + \frac{2}{log(2+1)} + \frac{3}{log(3+1)} + \frac{0}{log(4+1)} = 3 + 1.262 + 1.5 + 0 = 5.762 \)
      • Cons - nDCG does not penalize irrelevant search result
  • Online
    • Click-through rate (CTR) = number of clicked images / total number of suggested images
    • Average daily/weekly/monthly time spent on suggested images

Deployment and serving

  • Indexing service
    • Indexes the embeddings of new images so that NN search can easily find it
  • Embedding generation service
    • Preprocess the input images and use trained model to determine its embedding
  • Re-ranking service
    • Filters inappropriate results, removes duplicates, etc
  • Nearest neighbor service
    • Retrieve similar images from embedding space
    • Exact NN
      • Time O(ND) where N is total number of points and D is dimension
      • N could be billions
    • Approximate NN
      • Time O(log(N)D)
      • Tree-based
      • Locality sensitive hashing (LSH)
      • Clustering-based
