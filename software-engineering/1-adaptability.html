# Everything changed unexpectedly

I was lead data engineer for TD Securities data management team
Our team owned data platform where users could ingest and consume data

In October 2025, I was tasked to provision self-service data platform
I had three other members to work with, project manager, front-end developer, and back-end developer
Platform consisted of internal app rewritten in Reach and Java, and infrasturcture on Databricks and Azure Data Lake Storage
Initally, I was given until Feburary 2026 deliever it
However, in the middle of Novemeber, my manager told me that I needed to complete it in two weeks

I suggested the team to make a number of trade-off to choose speed over tech debt
We initially designed micro-service API architecture, but I proposed that we go with monolithic to save time
I suggested the team to skip PR approvals and do direct merge
I suggested the team to bypass TD internal process of writing change order
Bypassing PR approval and change order saved two weeks of infra deployment effort, and I provisioned infra in the first week
Bypassing micro-service API architecture saved months of effort, and the team deployed API in the second week

We aniticipated about 40 internal users, but in reality about 150 users came to use our platform within the first month
The executive leadership team complemented our team's effort to enabling data platform very fast


# You had to pivot quickly

I was machine learning engineer at Layer6 AI, AI specialized org within TD Bank
My team was responsible for ML serving design, MLOps, and ML infrastucture

In December 2021, my manager reached out to me on Saturday
There was an industry-wide issue where a package that almost every Java application used contained a critical security risk
That package was called log4j and the risk allowed attacker to perform remote code execution (RCE)

I suggested the team to immediately shut down all running models in production
Our models were all batch models, whose inference result can be re-computed at any time
Impact of shutting down models is not producing inference results for only few days
I shut down all the containers, patch them with newer version of log4j, and redeployed them by Monday night

Our models were not exploited by the vulnerability
If we had API models, I would not have shut down the containers
Attacker must first compromise firewall to access resources in Vnet in cloud
Then, they need to break into rules set by network security groups
This is unlikely to happen
