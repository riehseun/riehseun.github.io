# You saw a need and stepped up

I was machine learning engineer in AI Platform team in Dec 2025
My team was responsible for ML serving design, MLOps, and ML infrastucture

My team was using custom built python packages to compute batch inference
These packages had dependencies, were were updated often for security patches
Newly built packages somethime broke code running in production
I wanted to run tests to detect potential failures

I created three pipelines in Azure Datafactory and connected them for automatic execution
First pipeline retrieves the run IDs of last inference and ground truth jobs generated by existing python package
Second pipeline runs new inference and ground truth jobs using updated python package
Third pipeline compares outputs from two pipelines and verifies checksum

Issues coming from dependency update were caught before any inference job runs in production


# You drove change across teams

I was working for cloud engineering in Mar 2020
My team was responsible for developing code and infrastructure pipeline for the Bank
My team had local development environment for developing pipeline

The setup of local development environment was completed manual
New developer took several weeks fully setup environment
There were always mistakes, each developer's environment became different

I decided to spent one month to automate environment setup
I wrote docker-compose file to deploy Docker containers for tools like BitBucket, Jenkins, XL Release, XL Deploy, and Postgres SQL
I wrote bash scripts to configure all these tools in fully automated way

New developer tooks few hours to setup environment


# You influenced without authority

I was working for DevOps team at TD Wealth in Mar 2019
My team was responsible for releasing code and infrastructure to production
My team was using pipeline developed by cloud engineering team to release code and infrastructure
My team didn't know the pipelined worked under the cover

Pipeline was very buggy and my team was chasing support people who are very unresponsive
I wanted my team to resolve bugs ourselves rather than having to rely on support team

Asked the team to select few members to inner-source to cloud engineering to work as pipeline developers
I went-in as the first member from our team
I worked for cloud engineering to develop, test, and release the pipeline code
I debugged the problems that my team was facing and shared the knowledge
Subsequently, another team member got chosen as the next person to become cloud engineer

Time to resolve pipeline issues went from days to few hours on average
