# You disagreed with teammates or leaders

I was leading initiative to provision data platform for TD Securities in Nov 2025
While I was working to deploy the platform into production, my manager demanded that we open up DEV environment to users first

I was converned about stability of user experience because we were constantly pushing changes to DEV environment as we were rolling out features
 
I asked my manger and my team members to open up a call so that we can talk in person
I first heard my manager, and he advocated that users are urgently waiting for something that they can play with
I aligned with my manager to do three things
  Only on-board first set of pilot users who are already familar with databricks
  Treat DEV like sandbox environment where data are not real data
  Limit users to play within their own catalog (logical separation of Unity Catalog in Databricks)

Initial set of users (about 5-10) were happy to play with DEV instances first
Users who were later on-boarded to production instance (there were about 150) gave great feedback to my team

# You had to navigate competing priorities

Supporting Enterprise DevOps pipeline (TD Wealth SDDE) (Mar 2021)
Inner-sourcing in cloud engineering, working as a developer for the enterprise pipeline
Users were always complaining about pipeline support
Users didn't know how to use the pipeline

Had to to perform the role of not only developer but release engineer and technical support at the same time

Created a FAQ document that summarized common problems and solutions
First redirected user to FAQ document when they reported problems
Set priorities such that production technical support first, development second, and all others third
Advised stakeholders to investigate any non-production issues themselves before contacting myself

Usage rate of deployment pipeline increased about twice in the bank about a year after I join the team
Helping users learn how to use the tools rather than solving problems on their behalf wins in long run
