Python basic

  • Python is translated to bytecode, and then to machine code before the code gets executed
  • Everything is object
  • Packages are generally directories
  • Modules are generally files
  • break keyword terminates the loop and jump to first statement after the loop
  • #!/usr/bin/env python3 identifies which interpreter is used for the program
  • Parameters are what the function expects
  • Arguments are actual parameters that are passed to the function
  • list.sort() only sorts list
  • sorted() sorts any iterables. Also, it creates a copy of original iterable

Operators

  • // division discarding fraction
  • % remainder
  • ** power
  • == value checks for equivalent content
  • is identity checks for same object
  • & bitwise AND
  • | bitwise OR

Code execution

  • No type checking until runtime (Dynamically typed)
  • Code gets compiled before getting executed
    • But there is no noticible compilation phase so we normally say it is interpreted language
  • Written in C language and runs natively
    • Other variations include Jython, which is written in Java and runs on JVM
  • REPL (Read -> Evaluate -> Print -> Loop)

Pass by object reference

  • Consider 
    a = [0,1]  # Variable "a" points to object "[0,1]"
  • If variable a is passed by reference, anything that function does to variable a also applies to object [0,1]
  • If variable a is passed by value, function gets copies of a and [0,1], which are stored in new location in the memory
  • Python is where object references are passed by value or pass by object reference. Function gets copy of a but that copy of a points to the same object in the memory

String

  • Immutable
    • Can be used as dictionary keys or set elements
    • Concatenation
      • This is inefficient due to new instance being created each time 
        for substring in substrings:
    string += substring
      • Use this instead 
        string = "".join(substrings)
      • If number of substrings to concatenate is known, use f-strings
  • String can only store unicode text
  • String needs to be encoded to binary before saved to disk or sent over network
    • Encode string
      • str.encode(encoding, errors)
        • encoding is by default utf-8
        • errors is by default strict and can be ignore, replace, etc
      • bytes(source, encoding, errors)
        • source should be str
    • Decode string
      • bytes.decode(encoding, errors)
      • str(source, encoding, error)
        • source should be byte

Python advanced syntax

Interator

  • Iterable - object that has _iter_ method defined
  • Iterator - object that has both _iter_ and _next_ methods defined
    • _iter_ returns iterator object itself
    • _next_ returns the next element

Generator

  • Function with yield statements
    • You can iterate only once (Values are not stored in memory)
  • Return intermediate results
  • Used when streaming data buffers
def fibonacci():
    a, b = 0, 1
    while True:
        yield b
        a, b = b, a + b

fib = fibonacci()
next(fib)  # 1
next(fib)  # 1
next(fib)  # 2
[next(fib) for i in range(10)]  # [3, 5, 8, 13, 21, 34, 55, 89, 144, 233]

Decorator

  • Function that receives a function and returns an enhanced one
def mydecorator(function):
    def wrapped(*args, **kwargs):
        # do some stuff before the original
        # function gets called
        result = function(*args, **kwargs)
        # do some stuff after function call and
        # return the result
        return result
    # return wrapper as a decorated function
    return wrapped

Context managers

  • Use of with keyword
# Bad
hosts = open('/etc/hosts')
try:
    for line in hosts:
        if line.startswith('#'):
            continue
        print(line.strip())
finally:
    hosts.close()
# Good
with open('/etc/hosts') as hosts:
    for line in hosts:
        if line.startswith('#'):

PEP 8

  • PEP 8 is a guideline for best practices in writing Python code

Naming convention

  • Constants
    • Uppercase with underscore
  • Public and private variables, functions and methods, arguments, properties
    • Lowercase with underscore
  • Class or instance variables
    • Leading underscore
  • Special methods
    • start and end with double underscore
  • Classes and exceptions
    • CamelCase
  • Base and abstract classes should be named as
    • BaseClassname
    • AbstractClassname
  • Boolean variables should start with is or has
  • Collections (list, dictionary, tuple, set) should be named with plurals
  • Use explicit names for dictionaries

Indentation

  • Use 4 spaces per indention level
  • Observe "hanging" indentation
# Good
foo = long_function_name(var_one, var_two,
                         var_three, var_four)

# Good
# Hanging indent example. (No arguments in the first line of the function)
var = function(
    arg_one, arg_two,
    arg_three, arg_four)

# Bad
foo = long_function_name(var_one, var_two,
    var_three, var_four)
# Good
def long_function_name(
        var_one, var_two, var_three,
        var_four):
    print(var_one)

# Bad
def long_function_name(
    var_one, var_two, var_three,
    var_four):
    print(var_one)

Braces and brackets

    4, 5, 6,

Imports

  • Should be on separate lines.
  • Order of imports should be (also there should be line breaks between each type of import)
    • Standard library imports
    • Related third party imports
    • Local application/library specific imports
# Good
import os
import sys
from subprocess import Popen, PIPE

# Bad
import sys, os

Wildcard imports should be avoided

# Good
import math
from math import pi

# Bad
from math import *

Quotes

  • Single quote and double quotes are the same. Make a choice and stick to it

Whitespaces

  • Avoid unnecessary whitespace
# Good

# Bad

# Good

# Bad

# Good

# Bad

y             = 2

Always surround these binary operators with a single space on either side

  • Assignment (=)
  • Augmented assignment (+=, -=, etc)
  • Comparisons (==, <, >, !=, <>, <=, >=, in, not in, is, is not)
  • Booleans (and, or, not)

Comments

  • Don't use inline comments to explain obvious fact
  • Leave at least two spaces after the statement when making inline comments

Polymorphism

  • Same object exhibiting different forms and behaviors
  • For example, children classes inherits the same parent class but implements the method from parent differently (Method overriding)

Operator overloading

  • When + operator is called, __add__ function is invoked in Python
  • We can overload built-in methods like __add__ and __sub__
class Com:
    def __init__(self, real=0, imag=0):
        self.real = real
        self.imag = imag

    def __add__(self, other):  # overloading the `+` operator
        temp = Com(self.real + other.real, self.imag + other.imag)
        return temp

    def __sub__(self, other):  # overloading the `-` operator
        temp = Com(self.real - other.real, self.imag - other.imag)
        return temp

Context manager

  • Open files, write to them, and close them
from contextlib import contextmanager

@contextmanager
def file_open(path):
    try:
        f_obj = open(path, 'w')
        yield f_obj
    except OSError:
        print("We had an error!")
    finally:
        print('Closing file')
        f_obj.close()

if __name__ == '__main__':
    with file_open('test.txt') as fobj:
        fobj.write('Testing context managers')

Operating system

Program

  • An executable file that has a set of instructions and associated data
  • OS kernel invoke process which is an environment where program runs

Process

  • Running instance of a program and is an isolated execution entity that does not share data
  • Consists of instructions, user data, system data segments, CPU. memory, address space, disk, network I/O required at runtime
  • Creating process requires a separate system call

Thread

  • Thread is smallet execution unit in a process
  • It can be managed by a scheduler
  • Thread benefit
    • Higher throughput
      • This is possible only in multicore environment
    • Responsive applications
    • Efficient utilization of resources
      • Thread creation is lightweight compared to spawning new process
      • All threads within a process share the same address space
  • Thread challenges
    • Bugs can't be reproduced consistently
    • Code maintenance becomes harder
    • Consumes system resources
      • Creating thread consume memeroy and CPU cycles for book-keeping
    • Slows down programs
      • Acquiring and releasing locks adds to program execution time

Concurrency

  • Ex. single core
  • One program can be suspended while the other executes

Parallelism

  • Ex. multi core
  • Ability to execute multiple programs at the same time

Preemptive multitasking

  • OS scheduler decides which thread or program gets CPU next and for how long

Throughput

  • How much work is done in unit time
  • Ex. number of files processes in one minute

Latency

  • Time required to complete a task
  • How much time taken to process all files

CPU bound

  • High utilization of CPU
  • Ex. image processing, matrix multiplication

I/O bound

  • Wait for input or output operations while CPU sits idle

Critical section

  • Code that can be executed by more than one thread

Race condition

  • Threads access shared resource/variable to cause inconsistent application data

Deadlock

  • First thread holds resource required by second, second thread holds resouce required by first

Livelock

  • First thread passes required resouce to second, second thread passes required resouce to first. This continuously happens in a loop

Starvation

  • Greedy threads continuously hog shared system resource

Semaphore

  • Limited number of permits to give out
  • Semaphore can act as mutex when the number of permit is 1
    • In the context of mutex, same thread must acquire and release the mutex
    • In the context of semaphore, different threads can acquire and release semaphore

Monitor

  • Mutex plus one or more condition variable
  • wait() - when called on condition variable, release mutex and put calling thread into a queue
  • signal() - when called on condition variable, make a thread in a queue ready for execution

RLock (Reentrant lock)

  • Lock that can be reacquired by the same thread

Daemon thread

  • Runs in background
  • Python program does not exit until all threads terminate, however it exits even when daemon thread is running

Multi-processing

  • Fork
  • Spawn
    • Fork followed by exec
  • Fork-server

Async.io

  • High concurrency using a single thread or event loop
