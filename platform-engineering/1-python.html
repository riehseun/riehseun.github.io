Python Basic

  • Python is translated to bytecode, and then to machine code before the code gets executed.
  • Everything is object.
  • Packages are generally directories.
  • Modules are generally files.
  • break keyword terminates the loop and jump to first statement after the loop.
  • #!/usr/bin/env python3 identifies which interpreter is used for the program.
  • Parameters are what the function expects.
  • Arguments are actual parameters that are passed to the function.
  • list.sort() only sorts list.
  • sorted() sorts any iterables. Also, it creates a copy of original iterable.

Code execution

  • No type checking until runtime. (Dynamically typed)
  • Code gets compiled before getting executed.
    • But there is no noticible compilation phase so we normally say it is interpreted language.
  • Written in C language and runs natively.
    • Other variations include Jython, which is written in Java and runs on JVM.
  • REPL (Read -> Evaluate -> Print -> Loop)

Pass by object reference

  • Consider 
    a = [0,1]  # Variable "a" points to object "[0,1]"
  • If variable a is passed by reference, anything that function does to variable a also applies to object [0,1].
  • If variable a is passed by value, function gets copies of a and [0,1], which are stored in new location in the memory.
  • Python is where object references are passed by value or pass by object reference. Function gets copy of a but that copy of a points to the same object in the memory.

Method overload and overriding

  • Overloading: same function name but different signiture within the same class.
  • Overrideing: same function name and signigure between parent and child class.
  • Python does not support method overloading. (If overloading, Python uses the latest method)

Yield

  • Iterables: something you can interate like list and string.
  • Generators: something you can iterate only once. (Values are not stored in memory)
  • yield: similar to return, but returns a generator.

Class method vs static method

  • The methods use different decorators @classmethod and @Staticmethod
  • Class method takes class as parameter while static method does not.
  • Class method are accessible using the class name while static method are utility functions inside the class.

Decorator

  • Decorator is outer function, which takes in an original function as argument and returned modified version.
def smart_divide(func):
    def inner(a, b):
        print("I am going to divide", a, "and", b)
        if b == 0:
            print("Whoops! cannot divide")
            return

        return func(a, b)
    return inner

@smart_divide
def divide(a, b):
    print(a/b)

divide(2,5)

divide(2,0)

PEP 8

  • PEP 8 is a guideline for best practices in writing Python code.

Indentation

  • Use 4 spaces per indention level.
  • Observe "hanging" indentation.
# Good
foo = long_function_name(var_one, var_two,
                         var_three, var_four)

# Good
# Hanging indent example. (No arguments in the first line of the function)
var = function(
    arg_one, arg_two,
    arg_three, arg_four)

# Bad
foo = long_function_name(var_one, var_two,
    var_three, var_four)
# Good
def long_function_name(
        var_one, var_two, var_three,
        var_four):
    print(var_one)

# Bad
def long_function_name(
    var_one, var_two, var_three,
    var_four):
    print(var_one)

Braces and brackets

my_list = [
    1, 2, 3,
    4, 5, 6,
]

Imports

  • Should be on separate lines.
  • Order of imports should be (also there should be line breaks between each type of import)
    • Standard library imports
    • Related third party imports
    • Local application/library specific imports
# Good
import os
import sys
from subprocess import Popen, PIPE

# Bad
import sys, os

Wildcard imports should be avoided

# Good
import math
from math import pi

# Bad
from math import *

Quotes

  • Single quote and double quotes are the same. Make a choice and stick to it.

Whitespaces

  • Avoid unnecessary whitespace.
# Good
foo = (0,)

# Bad
bar = (0, )
# Good
spam(ham[1], {eggs: 2})

# Bad
spam( ham[ 1 ], { eggs: 2 } )
# Good
x = 1
y = 2
long_variable = 3

# Bad
x             = 1
y             = 2
long_variable = 3

Always surround these binary operators with a single space on either side.

  • Assignment (=)
  • Augmented assignment (+=, -=, etc)
  • Comparisons (==, <, >, !=, <>, <=, >=, in, not in, is, is not)
  • Booleans (and, or, not)
# Good
i = i + 1
submitted += 1
x = x*2 - 1
hypot2 = x*x + y*y
c = (a+b) * (a-b)

# Bad
i=i+1
submitted +=1
x = x * 2 - 1
hypot2 = x * x + y * y
c = (a + b) * (a - b)
# Good
def complex(real, imag=0.0):
    return magic(r=real, i=imag)

# Bad
def complex(real, imag = 0.0):
    return magic(r = real, i = imag)

Line length

  • Limit all lines to maximum 79 characters. (72 for docstrings and comments)
  • This is for human readability. (Some devices can still render only upto 80 chars at a time)

Comments

  • Comments should be complete sentences with first character capitalized.
  • Don't use inline comments to explain obvious fact.
  • Leave at least two spaces after the statement when making inline comments.

Python data types

List

  • Dynamically sized array. Implements a stack.
  • Allows duplicates.
  • Mutable.
  • Ordered.
  • List is much heavier than array. Each list contains block of pointers, which in turn points to Python objects.
# Create an empty list.
my_list = []

# Add an element at the end of a list.
my_list.append(element)  # O(1)

# Add an element at a specific position of a list.
my_list.insert(index, element)  # O(n)

# Remove the last element from a list.
my_list.pop()  # O(1)

# Remove by item. (It will remove only the first match!)
my_list.remove(element)  # O(n)

# Remove by index.
del my_list[index]  # O(n)

# Search an element in a list.
element in my_list  # O(n)

List continued

# Get max or min of a list.
min(my_list)  # O(n)
max(my_list)  # O(n)

# Length of a list.
len(my_list)  # O(1) Python keeps this information as list grows in size.

# Get the last element of list.
my_list[-1]  # O(1)

# Slice.
my_list[a:b]  # O(b-a)

# Init an array with size 10.
my_list = [None]*10

# Sort list.
my_list.sort()  # O(nlogn)

# Reverse list.
my_list.reverse()  # O(n)

import random
# Pick a random element from a list.
random_element = random.choice(my_list)
# Pick a random number between two ranges (inclusive)
random_element = random.randint(a, b)

# List is mutable, so need.
new_list = my_list.copy()
new_list = my_list[:]

import bisect
# Add an item to a sorted array.
my_list = [1, 2, 4, 5]
bisect.insort(my_list, 3)

List of lists

# Sort list of lists by first element. (but by second element in reverse order)
my_list.sort(key = lambda x: (x[0], -x[1]))

# Get only first items in the lists of list.
[item[0] for item in my_list]

Dictionary

  • Can hold Key-value data types. Implements a hash table.
  • Allows duplicates. (But keys cannot be duplicate)
  • Mutable.
  • Unordered.
# Create an empty dictionary.
my_dict = {}

# "defaultdict": when a key does not exist, a new entry is created rather than keyError thrown.
from collections import defaultdict
my_dict = defaultdict(int)
my_dict = defaultdict(list)
my_dict = defaultdict(set)

# Add an element to a dictionary.
my_dict[key] = value  # O(1)

# Remove an element from a dictionary.
del dict[key]  # O(1)

# Search by a key in a dictionary.
key in my_dict  # O(1)

Dictionary continued

# Get items in the dictionary by the index.
list(my_dict.keys())[0]

# Iterate dictionary.
for key, val in my_dict.items():  # O(n)

# Get keys of dictionary.
my_dict.keys()   # O(1) in Python3 which returns "dict_keys" object. O(n) in python2 which returns "list" object.

# Get values of dictionary.
my_dict.values()  # O(1)

# Init a dictionary whose keys are 0,1,2 ... and values are empty lists.
{new_list: [] for new_list in range(10)}

import copy
# Dictionary is mutable, so need.
new_dict = copy.deepcopy(my_dict)

# Sort by value.
sorted(my_dict, key=my_dict.get)
sorted(my_dict, key=my_dict.get, reverse=True)

Set

  • Comma separated data types. Implements a hash table.
  • No duplicates.
  • Mutable.
  • Unordered.
# Create an empty set.
my_set = set()

# Add an element to a set.
my_set.add(element)  # O(1)

# Remove a random element from a set.
my_set.pop()  # O(1)

# Search an element in a set.
element in my_set  # O(1)

Set continued

# Check if a set is contained in another set.
set_A.issubset(set_B)  # O(n)

set_A.union(set_B)

set_A.intersection(set_B)

# set_A - set_B
set_A.difference(set_B)

set_A.update(set_B)

Tuple

  • Comma separate objects.
  • Allows duplicates.
  • Immutable.
  • Ordered.

String

# Split string by line break and store it in a list.
string.splitlines()

string.lower()

string.strip()

all_words = re.findall(r'\w+', string)

    yaml_dictionary = yaml.safe_load(file)

# Write yaml file.

    yaml.dump(yaml_dictionary, file)

# Write yaml file (Addressing list indentation problem).
# yaml.dump breaks indentation of list items.
class Dumper(yaml.Dumper):
    def increase_indent(self, flow=False, *args, **kwargs):
        return super().increase_indent(flow=flow, indentless=False)
print(yaml.dump(yaml_dictionary, file, Dumper=Dumper))
