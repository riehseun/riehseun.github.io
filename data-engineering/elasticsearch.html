Endpoint

# Check cluster health.
curl -k -u <username>:<password> -X GET https://<endpoint>:9200/_cluster_health
# Check license.
curl -k -u <username>:<password> -X GET https://<endpoint>:9200/_license

# Update licese. (With the license in "license.json" file)
curl -k -u <username>:<password> -X PUT https://<endpoint>:9200/_license -H "Content-Type: appliction/json" -d @license.json
# Create an API key.
curl -k -u <username>:<password> -X POST https://<endpoint>:9200/_security/api_key --header "Content-Type: application/json" --date '{}'

# Create an index.
apiKey=<encoded_api_key_from_above>
curl -k -H "Authorization: ApiKey $apiKey" -X PUT https://<endpoint>:9200/collections

# List all indices.
curl -k -u <username>:<password> -X GET https://<endpoint>:9200/_cat/indices

# Get an index.
curl -k -u <username>:<password> -X GET https://<endpoint>:9200/<index_name>

# Check types.
curl -k -u <username>:<password> -X GET https://<endpoint>:9200/<index_name>/_mapping

# Create a document.
apiKey=<encoded_api_key_from_above>
curl -k -H "Authorization: ApiKey $apiKey" -H "Content-Type: application/json" -X PUT https://<endpoint>:9200/collections/_doc/1 -d '<JSON>'

# Check a document.
curl -k -u <username>:<password> -X GET https://<endpoint>:9200/<index_name>/_doc/<document_id>

ECK

Client nodes

  • Deployment with autoscaler.
  • Acts as reverse proxy.
  • Queries master nodes.

Master nodes

  • Deployment with fixed replicas. (At least 3 for HA)
  • Holds metadata: data nodes, location, sharding, data replication.

Data nodes

  • StatefulSets where each instance has PVC.
  • Splits indices into shards and replicate information between nodes. (To prevent data loss)

Node role

Master

  • Creates/deletes index.
  • Tracks nodes in the cluster.
  • It is good idea to have nodes that are dedicated to master role only.

Data

  • data_hot: most recent and frequently searched data, fast read and write.
  • data_warm: typically holds data from recent weeks, updates are infrequent.
  • data_cold
  • data_frozen

Ingest

  • Executes pre-processing pipeline.
