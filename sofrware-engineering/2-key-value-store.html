Key-value store

  • Distributed hash table.
  • Key is generated by a hash function.
  • Value can be anything.

Requirement

Functional

  • Users can configure availability vs consistency.
  • Users can write into key-value store.

Non-functional

  • Availability
  • Scalability
  • Fault tolerance

API

  • get(key)
  • put(key, value)

Component design

Scalability

  • Use consistent hashing for scalable hardware.

Falut tolerance

  • Use primary-secondary or peer-to-peer for data replication.
