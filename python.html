Important concepts

  • Everything is object.
  • Numbers : Operators include // (division discarding fraction), % (remainder), ** (power)
  • Queue using list : inefficient. Instead use "from collections import deque" and deque.append(x) & deque.popleft()
  • Range : To use range(1,10) as a list, do list(range(1,10))
  • Types : "int", "float", "None", "bool" (True/False)
  • Relational Operators : "==", "!=", etc
  • "break" keyword terminates the loop and jump to first statement after the loop
  • Shebang : identify which interpreter is used for the program. For example, #!/usr/bin/env python3
  • Value - equivalent content ("==" operator). Identity - same object ("is" operator)
  • Arguments are passed by object-reference (mutable objects can be modified)
  • built-in types : int, float, str, list, dict
  • packages - generally directories
  • modules - generally files

PEP 8

PEP 8 is a guideline for best practices in writing Python code.

Use 4 spaces per indention level. Observe "hanging" indentation.

# Good
foo = long_function_name(var_one, var_two,
                         var_three, var_four)

# hanging indent example (no arguments in the first line of the function)
var = function(
    arg_one, arg_two,
    arg_three, arg_four)

# Bad
foo = long_function_name(var_one, var_two,
    var_three, var_four)
# Good
def long_function_name(
        var_one, var_two, var_three,
        var_four):
    print(var_one)

# Bad
def long_function_name(
    var_one, var_two, var_three,
    var_four):
    print(var_one)

Braces and brackets

my_list = [
    1, 2, 3,
    4, 5, 6,
]

Limit all lines to maximum 79 characters. (72 for docstrings and comments)

This is for human readability. (Some devices can still render only upto 80 chars at a time)

Watch the line break rule for operators.

# Good
income = (gross_wages
      + taxable_interest)

# Bad
income = (gross_wages +
      taxable_interest)

Line breaks

# Top level functions and class definition should have two line breaks.
# Methods inside class should have one line break.
# Line breaks inside function should be used sparcingly to group logics.


class MyClass(object):
     def method_1(self, arg1, arg2):

     def method_2(self, arg1, arg2):

Imports should be on separate lines.

# Good
import os
import sys
from subprocess import Popen, PIPE

# Bad
import sys, os

Order of imports should be (also there should be line breaks between each type of import)

  • Standard library imports
  • Related third party imports
  • Local application/library specific imports

Wildcard imports should be avoided

Single quote and double quotes are the same. Make a choice and stick to it.

Avoid unnecessary whitespace.

# Good
foo = (0,)

# Bad
bar = (0, )
# Good
spam(ham[1], {eggs: 2})

# Bad
spam( ham[ 1 ], { eggs: 2 } )
# Good
x = 1
y = 2
long_variable = 3

# Bad
x             = 1
y             = 2
long_variable = 3
# Always surround these binary operators with a single space on either side: assignment (=), augmented assignment (+=, -= etc.), comparisons (==, <, >, !=, <>, <=, >=, in, not in, is, is not), Booleans (and, or, not)

# Good
i = i + 1
submitted += 1
x = x*2 - 1
hypot2 = x*x + y*y
c = (a+b) * (a-b)

# Bad
i=i+1
submitted +=1
x = x * 2 - 1
hypot2 = x * x + y * y
c = (a + b) * (a - b)
# Good
def complex(real, imag=0.0):
    return magic(r=real, i=imag)

# Bad
def complex(real, imag = 0.0):
    return magic(r = real, i = imag)

Comments

Comments should be complete sentences with first character capitalized. Don't use inline comments to explain obvious fact. Leave at least two spaces after the statement when making inline comments.

Naming convention

  • Never use l, O, I as variable names.
  • Classname should be CapWords. (same convention should be used for Exceptions)
  • Functions and variables should be all lowercase separated by spaces.
  • Leading underscore is for non-public method only.
  • Constants should be all uppercase separated by underscores.

Python data types

# List
# Dynamically sized array. Implements a stack.
# Allows duplicates.
# Mutable.

# Create an empty list.
my_list = []

# Add an element at the end of a list.
my_list.append(element)  # O(1)

# Remove the last element from a list.
my_list.pop()  # O(1)

# Add an element at a specific position of a list.
my_list.insert(index, element)  # O(n)

# Remove an element from a specific position of a list.
del my_list[index]  # O(n)

# Get max or min of a list.
min(my_list)  # O(n)
max(my_list)  # O(n)

# Search an element in a list.
element in my_list  # O(n)

# Length of a list.
len(my_list)  # O(1) Python keeps this information as list grows in size.
# List continued
# Init an array with size 10.
array = [None]*10

# Remove an item from list.
# It will remove only the first match!
list.remove(element)

# Get the last element of list.
list[-1]

# Sort list.
list.sort()

# Reverse list.
my_list.reverse()  # O(n)

# Sort list of lists by first element (but by second element in reverse order)
list.sort(key = lambda x: (x[0], -x[1]))

import random
# Pick a random element from a list.
random_element = random.choice(list)
# Pick a random number between two ranges (inclusive)
random.randint(a, b)

# List is mutable, so often times need.
new_list = list.copy()
new_list = list[:]

import bisect
# Add an item to a sorted array.
list = [1, 2, 4, 5]
bisect.insort(list, 3)
# Dictionary
# Can hold Key-value data types. Implements a hash table.
# Allows duplicates. (But keys cannot be duplicate)
# Mutable.

# Create an empty dictionary.
my_dict = {}

# Add an element to a dictionary.
my_dict[key] = value  # O(1)

# Remove an element from a dictionary.
del dict[key]  # O(1)

# Search by a key in a dictionary.
key in my_dict  # O(1)
# Dictionary continued

# Get items in the dictionary by the index.
list(my_dict.keys())[0]

# Iterate dictionary.
for key, val in dict.items():

# Get keys of dictionary.
dict.keys()

# Get values of dictionary.
dict.values()

# Init a dictionary whose keys are 0,1,2 ... and values are empty lists.
{new_list: [] for new_list in range(10)}
# Set
# Comma separated data types. Implements a hash table.
# No duplicates.
# Mutable.

# Create an empty set.
my_set = set()

# Add an element to a set.
my_set.add(element)  # O(1)

# Remove a random element from a set.
my_set.pop()  # O(1)

# Search an element in a set.
element in my_set  # O(1)

# Comma separate objects. Implements a hash table.
# Allows duplicates.
# Immutable.

# Create an empty tuple.
my_tuple = ()

string.splitlines()

# Convert to lowercase.
string.lower()

# Remove spaces at the beginning and end.
string.strip()

all_words = re.findall(r'\w+', string)

string.index(substring)

if substring in string:

    yaml_dictionary = yaml.safe_load(file)

# Write yaml file.
with open("my_file.yaml", "w") as file:
    yaml.dump(yaml_dictionary, file)

# Write yaml file (Addressing list indentation problem).
# yaml.dump breaks indentation of list items.
class Dumper(yaml.Dumper):
    def increase_indent(self, flow=False, *args, **kwargs):
        return super().increase_indent(flow=flow, indentless=False)
print(yaml.dump(yaml_dictionary, file, Dumper=Dumper))
