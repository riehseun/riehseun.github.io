Elasticsearch cluster migration (Layer6 AI)

  • S
    • Database for storing inference and ground truth data called Elasticsearch
    • ES cluster was deployed in 3 virtual machines
  • P
    • ES was at outdated with version 6
    • Did not support authentication
    • Read/write was slow. Write was taking more than 3 hours during batch inference
    • Disk was nearly full
  • S
    • Upgraded ES version 6 to 8
    • Capacity estimate for the next 5 years
    • Deployed the cluster in kubernetes because model training and inference were also running in Kubernetes (I wanted native integration between them)
    • Higher compute and lower storage to hot nodes and lower compute and higher storage for cold nodes
    • Splited each ES index to many shards such that each shard did not exceed 600MB in order to acheive fast read
  • I
    • Twice faster in read/write
  • L
    • Official documentation may not be fully correct
    • In particular, taking snapshot of data from version 6 and restoring it in version 8 did not work. Thus, I had to leaverage reindexing
    • Learned to leave lots of lead time for implementation
